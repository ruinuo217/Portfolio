About

三種背景，一個視角

我的職涯路徑不是直線，而是弧線。高中念的是台北影視音實驗學校，從拍攝、打光、剪輯、導演到後期特效與調光，完整走過一部影片的製作流程——那段訓練讓我從一開始就習慣用視覺說故事。從 3D 渲染設計師，到 Shopee 行銷活動企劃，再到現在全力投入的 AI 應用工程師、前端工程師、數據分析師，每一步都是為了讓自己能同時從使用者、數據、工程三個角度思考問題。

在蝦皮兩年，最大的收穫不只是學會跑電商活動，而是養成用數字說話的習慣——每月客訴數怎麼降、漏斗在哪裡斷、改一個字會不會少被客訴，這些思考方式讓我很受用。不過說實話，讓我最有成就感的，不是活動數字漂亮，而是發現「欸如果按鈕移過去用戶應該會提高 CTR」的那個瞬間——那時我才意識到，我更想待在能把東西做出來的那一側。

設計師的同理心、行銷人的數據敏感度、加上工程師的系統思維，正是 AI 時代最稀缺的組合。

工安影像辨識
AI 告警系統

目的工地安全帽偵測仰賴人工抽查，事故難以預防；需要一套自動偵測、分級告警並能讓主管即時介入的系統。
成果從影像輸入到 LLM 每日摘要報告全程自動化；KMeans 自動校準信心度門檻取代人工設定；LangGraph 實作高低風險分流、3 次重試迴圈與 Human-in-the-Loop 主管審核機制，有效降低人工巡檢成本，確保告警可靠性，讓主管得以即時介入高風險事件。
技術1. 以 YOLOv8 訓練安全帽偵測模型，輸出結構化 Event JSON 進入規則引擎。
2. 用 sklearn KMeans 分群自動校準信心度門檻，取代人工設定。
3. 以 LangGraph StateGraph 實作高低風險分流、重試機制與 Human-in-the-Loop 主管審核。
4. 用 FastAPI 建立 REST API Server；以 Gemini / OpenAI 生成每日摘要報告。

台灣工程師職缺
競爭力分析系統

目的台灣工程師職缺資料分散於六大平台、格式不統一；同時需要在開發前期建立穩健的資料庫架構，避免後期因設計不良導致高重構成本。
成果主導 SA / SD 規劃，設計高擴充性 MySQL Schema；整合 6 大平台共 12,297 筆職缺，建立 GCP 雲端備援資料庫；Tableau 7 Story Points 呈現薪資分布、技能四象限等分析，為求職者與企業提供可即時查詢的工程師職缺競爭力分析平台。
技術1. 主導 SA / SD 規劃：設計 3 張 MySQL 資料表（jobs / skills / job_skills_mapping）、ERD 與 12 欄爬蟲規格，發布 DevOps SOP 供全員遵循。
2. 主導 ETL 資料清洗（CSV 合併去重 → MySQL 入庫），同步建立 GCP 雲端備援資料庫。
3. 主導 Tableau 敘事架構規劃，直連 MySQL 建置 Dashboard 框架，產出 7 個 Story Points 視覺化頁面。
4. 協助前端介面開發與視覺呈現。
