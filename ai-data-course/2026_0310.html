MongoDB 聚合分析與 Python 整合

3/10 課程筆記

聚合管道概念

什麼是聚合管道

觀念

Aggregation Pipeline = 資料處理框架。類比工廠生產線：原料(文件) → 加工站(階段) → 成品(結果)。

  • 資料庫層級完成計算，比應用程式端處理更高效
  • 可組合多個階段，每個階段的輸出是下一階段的輸入
  • 語法清晰，易於理解與維護

常用聚合階段

觀念

聚合管道的每個階段對應 SQL 的常見操作。

聚合階段功能對應 SQL
$match篩選文件WHERE
$group分組統計GROUP BY
$sort排序ORDER BY
$project選擇欄位SELECT
$limit限制數量LIMIT
$skip跳過筆數OFFSET
$count計數COUNT(*)

常用聚合函數

觀念

在 $group 階段中使用的聚合運算子。

聚合函數功能
$sum總和或計數
$avg平均值
$max最大值
$min最小值
$first第一筆
$last最後一筆
$push收集成陣列

聚合管道實作

$match 篩選

實作

類似 find() 的條件篩選，放在管道第一階段。

  • 範例：{"$match": {"評分": {"$gte": 4.0}}}
  • 最佳實踐：放在管道前面以利用索引
  • 減少後續階段需要處理的資料量

$group 分組統計

實作

語法：{"$group": {"_id": "$分組欄位", "統計欄位": {"聚合函數": ...}}}

  • 統計各類型餐廳數量：_id: "$類型", 餐廳數量: {"$sum": 1}
  • 計算各類型平均評分：平均評分: {"$avg": "$評分"}
  • 全體統計（不分組）：_id: null，搭配 $sum、$avg、$max、$min
Tip: _id 中的 $ 表示引用欄位值；$sum: 1 表示每筆計數為 1。

$sort + $project

實作

排序與欄位選擇，美化輸出結果。

  • $sort：1 = 升序，-1 = 降序
  • $project：選擇欄位（1 顯示 / 0 隱藏）
  • 重新命名："餐廳類型": "$_id"
  • 計算新欄位：使用 $multiply、$sqrt 等運算子

多階段管道組合

實作

完整範例：找出評分最高的餐廳類型，5 個階段串接。

  • Stage 1 $match：篩選評分 ≥ 4 分
  • Stage 2 $group：按類型分組，計算平均評分、數量、總評論數
  • Stage 3 $sort：按平均評分降序排列
  • Stage 4 $limit：取前 3 名
  • Stage 5 $project：美化欄位名稱

$unwind 展開陣列

觀念

將陣列展開成多筆文件，常搭配 $group 做統計。

  • 範例：{"推薦菜色": ["滷肉飯", "牛腩", "青菜"]} 展開為 3 筆文件，各含一道菜
  • 用途：統計所有推薦菜色出現次數

練習：聚合管道

練習
優秀回答：聚合管道的關鍵理解：(1) 每個階段的輸出是下一階段的輸入，像工廠生產線 (2) $match 放前面利用索引提升效能 (3) $group 的 _id 用 $ 引用欄位值，聚合函數也要用 $ 引用欄位 (4) _id: null 表示不分組統計全體 (5) $unwind 把一筆含陣列的文件拆成多筆，常搭配 $group 做統計。對應 SQL：$match=WHERE, $group=GROUP BY, $sort=ORDER BY, $project=SELECT, $limit=LIMIT。常見錯誤：$group 中欄位忘記加 $、$avg 寫成 'rating' 而非 '$rating'。

Compass 聚合管道操作

Compass Aggregations 功能

實作

使用 Compass 的圖形介面建立聚合管道。

  • 點選 Aggregations 標籤
  • 點選 New Pipeline
  • 點選 Add Stage → 選擇階段類型 → 輸入參數
  • 點選 Run 執行管道
  • 每新增一個階段，右側即時顯示結果，方便逐步驗證

地理位置查詢

地理位置查詢概念

觀念

MongoDB 原生支援地理位置查詢，適用於 LBS 應用。

  • 用途：查詢附近餐廳、某範圍內地點、計算距離
  • 台北座標範圍：經度 121.4~121.6，緯度 24.9~25.2

建立地理索引

實作

兩步驟建立地理查詢所需的索引。

  • 步驟 1：調整座標格式，新增「位置」欄位（GeoJSON 格式：type: "Point", coordinates: [Longitude, Latitude]）
  • 使用 updateMany + 聚合管道 $set 批次更新
  • 步驟 2：建立 2dsphere 索引：createIndex({"位置": "2dsphere"})
  • 也可在 Compass 的 Indexes 標籤建立
Tip: 座標順序是 [經度, 緯度] 不是 [緯度, 經度]！

$near 附近查詢

實作

查詢指定座標附近一定距離內的文件。

  • 語法：{"位置": {"$near": {"$geometry": {"type": "Point", "coordinates": [經度, 緯度]}, "$maxDistance": 距離(公尺)}}}
  • 範例：台大附近 1000m — coordinates: [121.5398, 25.0173]
  • $near 自動按距離排序，配合 limit(3) 取最近 3 間

練習：地理位置查詢

練習
優秀回答：地理位置查詢要點：(1) 必須先建立 2dsphere 索引，否則報錯 (2) 座標格式是 GeoJSON：type: 'Point', coordinates: [經度, 緯度]——注意是經度在前！(3) $maxDistance 單位是公尺（1000 = 1 公里）(4) $near 自動按距離排序，不需額外 sort (5) 可組合地理查詢與普通查詢：同時篩選附近 + 評分 + 類型。台北 101 座標 [121.5645, 25.0340] 附近 500m 查詢，配合評分 ≥ 4 和類型篩選，就是美食外送平台的推薦邏輯。

業界實例

美食外送平台推薦系統

觀念

推薦邏輯：距離 2000m 內 + 評分 ≥ 4 + 評論 ≥ 50 + 按綜合指標排序。

  • Stage 1 $geoNear：地理篩選 + 計算距離
  • Stage 2 $match：評分 / 評論篩選
  • Stage 3 $addFields：計算綜合指標 = 評分 × √評論數
  • Stage 4 $sort：綜合指標降序
  • Stage 5 $limit：前 10 名
  • Stage 6 $project：格式化輸出

練習：Dashboard

練習
優秀回答：Atlas Dashboard 是內建視覺化工具，優勢：(1) 直接從資料庫讀取，不需匯出資料 (2) 圖表即時更新，資料變動自動反映 (3) 可分享給團隊成員。建立了三種圖表：長條圖看各類型餐廳數量分布、圓餅圖看價格區間佔比（$$ 最多）、散佈圖看評分與評論數的關係（高評分不一定高評論數）。Dashboard 適合快速原型驗證和管理報告，但複雜分析還是需要 Python + 聚合管道。

Python 整合 — PyMongo

PyMongo 基礎

觀念

Python 操作 MongoDB 的官方套件。

  • 安裝：pip install pymongo
  • 連線：MongoClient(連線字串)
  • 選資料庫：client["food_map"]
  • 選集合：db["restaurants_raw"]

Python CRUD 操作

實作

PyMongo 提供與 mongosh 語法幾乎一致的方法。

  • insert_one() / insert_many()
  • find() / find_one()
  • update_one() / update_many()
  • delete_one() / delete_many()
  • 範例：for doc in collection.find({"評分": {"$gt": 4}}): print(doc["店名"])

Python 聚合管道

實作

使用 collection.aggregate([pipeline]) 執行聚合管道。

  • 範例：統計各類型平均評分
  • pipeline = [{"$group": {"_id": "$類型", "avg": {"$avg": "$評分"}}}, {"$sort": {"avg": -1}}]
  • 聚合管道在 Python 中就是一個 list of dict

練習：Python 整合

練習
優秀回答：PyMongo 操作與 Compass/mongosh 的語法幾乎一致，差異：(1) Python 用字典 {} 而非 JSON (2) 方法名用底線分隔如 insert_one 而非 insertOne (3) find() 回傳的是 Cursor 物件，需要用 for 迴圈或 list() 取出。連線字串中的密碼如果有特殊字元需要 URL encode。最佳實踐：連線字串存在環境變數或 .env 檔案，不要寫死在程式碼中。聚合管道在 Python 中就是一個 list of dict，與 mongosh 的寫法對應直接。

整合思考題

練習

假設你要為學校建立一個「校園美食推薦系統」，請設計完整的 MongoDB 方案。

優秀回答：(1) 資料庫設計：food_map 資料庫含 restaurants（餐廳）、users（使用者）、reviews（評論）三個 Collection。restaurants 使用嵌入式設計存基本資訊 + 位置 + 評分，reviews 獨立 Collection 存大量評論資料（避免文件超過 16MB）。(2) 資料來源：匯入政府開放資料作為基底，用 updateMany + $set 批次增強店名/類型/評分等欄位。(3) 核心功能：用 2dsphere 索引 + $near 實現「附近餐廳」；用聚合管道 $group + $sort 做「熱門排行榜」；用 $elemMatch 查詢特定評分的評論。(4) 技術棧：MongoDB Atlas 免費方案 + Python PyMongo 後端 API + 靜態前端。(5) 成本：Atlas M0 免費 512MB 足夠存數萬筆餐廳資料，Python 部署到 Cloud Run 閒置 $0。
