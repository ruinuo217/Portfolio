多模態 AI 與 Gemini 實戰

3/31 課程筆記 — 多模態輸入、Gemini 圖片分析與文件摘要、Structured Output、response_schema 與 Pydantic

多模態概念

什麼是多模態

觀念

單模態：文字 → 模型 → 文字。多模態：文字 + 圖像 + 影片 + 音訊 → 模型 → 文字 / 結構化資料。多模態模型能同時理解多種輸入格式，大幅擴展 AI 的應用場景。

模型支援輸入特點
GPT-4V文字 + 圖像強大但貴
Gemini文字 + 圖像 + 影片 + 音訊最全面
Claude文字 + 圖像長文本強
LLaVA文字 + 圖像開源

Gemini 基礎

Gemini 安裝與設定

實作

安裝 Google Generative AI SDK 並設定 API Key。

# 安裝套件
pip install google-generativeai

# 設定環境變數
export GOOGLE_API_KEY="your-key"
  • 取得 Key：makersuite.google.com/app/apikey
  • 支援格式：文字 / 圖片（JPG、PNG）/ PDF / 影片（MP4）/ 音訊（MP3、WAV）

基本文字問答

實作

使用 Gemini 進行最基本的文字生成。

import google.generativeai as genai

model = genai.GenerativeModel("gemini-2.5-flash")
response = model.generate_content("什麼是PPE?")
print(response.text)

圖像分析

實作

將圖片與文字提示一起送入模型，讓 Gemini「看懂」圖片。

from PIL import Image

image = Image.open("workplace.jpg")
response = model.generate_content([
    "描述這張工地照片的安全狀況", image
])
print(response.text)
有效提示技巧：具體明確（列出人員、檢查 PPE、指出隱患）遠比「描述這張圖片」效果好。具體 prompt 會得到結構化分析，模糊 prompt 只會得到泛泛描述。

Workshop：圖像分析

練習

設計 PPE 檢查 prompt，體驗多模態的實際價值。

解答：設計 PPE 檢查 prompt：「這是一張工地照片。請：1. 列出畫面中所有人員 2. 檢查每人是否配戴安全帽、反光背心 3. 指出任何安全隱患 4. 用表格格式輸出」。具體提示比模糊提示好 10 倍——「描述圖片」會得到泛泛描述，「檢查 PPE 配戴」會得到結構化分析。多模態的價值：監控攝影機的圖片不是文字，傳統 NLP 處理不了，Gemini 可以直接「看懂」圖片並輸出結構化結果。

文件輸入與摘要

PDF 處理

實作

透過 genai.upload_file() 上傳文件，讓 Gemini 讀取並摘要。

pdf_file = genai.upload_file("regulation.pdf")
response = model.generate_content([
    "摘要這份法規的PPE規定", pdf_file
])
print(response.text)
  • 支援格式：PDF / Word / 圖片 / 影片 / 音訊

Workshop：文件摘要

練習

上傳 PDF 後使用 Gemini 進行文件理解與摘要。

解答：上傳 PDF 後 Gemini 可以：(1) 摘要重點 (2) 提取特定條文 (3) 回答關於文件的問題。prompt 設計：「請摘要這份工安法規的重點，特別是關於 PPE 的規定，用條列式輸出」。注意：Gemini 會如實說明文件內容——如果 PDF 不是工安法規（如範例中的專題計畫書），它會明確指出而非硬編。這反而證明模型「沒有幻覺」。

Structured Output（結構化輸出）

prompt-based vs schema-based

觀念

兩種方式讓 LLM 輸出 JSON，但可靠度天差地別。

方式做法成功率格式保證適用場景
prompt-basedprompt 寫「只輸出 JSON」80-90%無快速測試
schema-based設定 response_schema100%Gemini 強制遵守正式系統
重點：跑 10 次有 1-2 次 json.loads() 爆掉 → 正式系統不能接受 prompt-based。

response_schema 用法

實作

透過 schema 定義（type / properties / required / enum），強制 Gemini 輸出符合格式的 JSON。

import json

schema = {
    "type": "object",
    "properties": {
        "person_count": {"type": "integer"},
        "violations": {"type": "array", "items": {
            "type": "object",
            "properties": {
                "type": {"type": "string"},
                "severity": {"type": "string"}
            }
        }}
    },
    "required": ["person_count", "violations"]
}

model = genai.GenerativeModel(
    "gemini-2.5-flash",
    generation_config=genai.GenerationConfig(
        response_mime_type="application/json",
        response_schema=schema
    )
)

response = model.generate_content(["分析這張照片的安全狀況", image])
data = json.loads(response.text)  # 一定成功

Pydantic 定義 Schema

實作

用 Pydantic BaseModel 定義 schema，可與 FastAPI 共用同一套 model。

from pydantic import BaseModel
from typing import List

class Violation(BaseModel):
    type: str
    severity: str
    description: str

class SafetyReport(BaseModel):
    person_count: int
    violations: List[Violation]

# 直接把 Pydantic model 當 schema 傳給 Gemini
方式定義方式型別檢查適用場景
dict schema手寫 dict無簡單結構
PydanticPython class自動驗證 + IDE 支援正式系統

Workshop：結構化輸出

練習

實作 schema-based 結構化輸出，比較 prompt-based 的可靠度差異。

解答： 
schema = {
    'type': 'object',
    'properties': {
        'person_count': {'type': 'integer'},
        'violations': {'type': 'array', 'items': {
            'type': 'object',
            'properties': {
                'type': {'type': 'string', 'enum': ['no_helmet', 'no_vest', 'other']},
                'severity': {'type': 'string', 'enum': ['high', 'medium', 'low']}
            },
            'required': ['type', 'severity']
        }},
        'recommendations': {'type': 'array', 'items': {'type': 'string'}}
    },
    'required': ['person_count', 'violations']
}

model = genai.GenerativeModel('gemini-2.5-flash',
    generation_config=genai.GenerationConfig(
        response_mime_type='application/json',
        response_schema=schema))
跑 10 次比較：prompt-based 約 8/10 成功（2 次多了 markdown 標記導致 json.loads 失敗），schema-based 10/10 成功。正式系統必須用 schema-based。enum 限制值的範圍，required 確保必填——跟 FastAPI 的 Pydantic 一模一樣。

HuggingFace 文字生成（進階）

AutoModel 生成

實作

使用 HuggingFace Transformers 的 AutoModel 進行本地文字生成。

from transformers import AutoModelForCausalLM, AutoTokenizer

tokenizer = AutoTokenizer.from_pretrained("gpt2")
model = AutoModelForCausalLM.from_pretrained("gpt2")

inputs = tokenizer("AI is transforming", return_tensors="pt")
outputs = model.generate(
    **inputs,
    max_length=100,
    temperature=0.7,
    do_sample=True,
    num_return_sequences=3
)

for i, output in enumerate(outputs):
    print(f"--- 序列 {i+1} ---")
    print(tokenizer.decode(output, skip_special_tokens=True))

Workshop：生成參數比較

練習

比較不同 temperature 與 do_sample 設定對生成結果的影響。

解答：
  • temperature=0.1：輸出保守重複，每次幾乎相同
  • temperature=1.5：輸出多樣發散，每次不同但可能不連貫
  • do_sample=False：每次完全相同（deterministic），適合需要一致性的場景
  • do_sample=True：有隨機性，適合需要變化的場景
實務選擇：分類/擷取 → temperature=0 + do_sample=False；創意寫作 → temperature=0.7~1.0 + do_sample=True。
