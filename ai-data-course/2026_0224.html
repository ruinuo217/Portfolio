Power BI 簡介與市場定位

Gartner BI 平台報告

觀念

Gartner 魔力象限將 BI 平台分為四個象限，Microsoft Power BI 位居領導者 (Leaders) 象限，在執行能力與前瞻性皆表現突出。

象限說明代表廠商
領導者執行力強、前瞻性高Microsoft (Power BI)
挑戰者執行力強、前瞻性較低-
遠見者前瞻性高、執行力較低Salesforce (Tableau)
利基者投入特定領域GoodData, Tellius 等
MAD Landscape：在 Machine Learning, AI and Data 生態圖中，Power BI 同時出現在 BI Platforms 與 Visualization 分類，且在 BI & AI Analytics Platforms 中佔有一席之地（Microsoft Azure: Power BI + Copilot）。

Power BI 優勢

重點
  • 免費下載與使用：部分功能無法使用，但基本功能已足夠學習與開發
  • Power Query 整合資料方便：內建強大的資料清洗與轉換引擎
  • 與 Microsoft 生態系統無縫整合：Excel、Azure、SQL Server 等
  • 支援本地及雲端部署：免費版不支援雲端發布
Power BI 缺點
  • 大數據處理性能瓶頸：硬體效能不足時會受影響
  • 複雜計算的學習曲線：DAX 語言學習較難，計算式、量值和高級數據建模需要技術背景
  • 整合性限制：非 Microsoft 生態系統整合較弱
  • 環境限制：僅支持 Windows 系統、消耗記憶體較大

Power BI vs Tableau

比較
功能Power BI (微軟)Tableau (Salesforce)
數據整合 內建 Power Query 處理雜亂數據極強，與 Azure、SQL Server 無縫連接 擅長將不同來源的巨量資料（Snowflake, BigQuery）快速關聯，效能更優
視覺化 實用導向，樣式較標準化，適合快速製作企業 KPI 看板 藝術級自由度，圖表精美、互動性極高，適合數據敘事
AI 功能 Copilot 全面嵌入，能用自然語言寫 DAX、自動生成報表 Tableau Pulse / Agent，主打主動推送洞察與異常通知
易用性 低難度上手，介面與 Excel 高度相似 中等學習難度，需理解維度與量值的拖拉邏輯
適用對象 中小企業、微軟生態系用戶 專業數據分析師、追求頂尖視覺效果的大型企業

1. 匯入資料 / 資料整理 (Power Query)

匯入資料流程

實作

練習資料包含 Covid-19 確診/死亡/康復資料、台灣各縣市感染個案、死亡統計及產品銷售等。

資料來源類型
  • Excel 活頁簿：Coronavirus data_20200404.xlsx（含 confirmed、Death、recovered 三個工作表）
  • 文字/CSV：疾管署_台灣各縣市新型冠狀病毒感染個案_20200407.csv
  • 其他來源：SQL Server、Web、JSON、資料夾等多種連接器
匯入步驟
  • 常用 → 取得資料 → 選擇資料來源類型
  • 在導覽器中勾選要匯入的資料表
  • 選擇「載入」直接載入，或「轉換資料」進入 Power Query 編輯器

Power Query 編輯器

核心

Power Query 是 Power BI 的資料清洗引擎，所有變更步驟都會記錄在右側「套用的步驟」中，可隨時回溯。

區域說明
左側：資料源列出所有已匯入的查詢（資料表），如 confirm、Death、recovered
中間：資料預覽顯示目前資料表的內容，可直接操作欄位
右側：查詢設定名稱、所有屬性，以及所有變更紀錄（套用的步驟）
常用操作
  • 使用第一個資料列作為標頭：將第一列提升為欄位名稱
  • 移除資料行：選擇不需要的欄位，右鍵移除或使用「選擇資料行」保留需要的
  • 移除空白資料列：移除資料列 → 移除空白資料列
  • 變更類型：點擊欄位標頭的型態圖示，可選擇小數、整數、日期/時間、文字、True/False 等
  • 取消資料行樞紐：將寬表轉換為長表格式（適合日期欄位展開）

欄位操作與分割

實作

右鍵點擊欄位標頭可進行多種操作：

  • 複製 / 移除：複製欄位或移除不需要的欄位
  • 分割資料行：依分隔符號、依字元數、依位置分割
    • 分割方向：最左邊一次、最右邊一次、一再重複
  • 變更類型：小數、整數、百分比、日期/時間等
  • 轉換：大小寫轉換、格式變更
  • 取代值：批次替換特定內容
  • 分組依據：依欄位值分組彙總
  • 填滿：向上/向下填充空白值
日期、時間轉換功能

選擇日期型態欄位後，可進一步擷取年、月、季、週、日等時間維度資訊。

DAX (Data Analysis Expressions)

DAX 基本規則

觀念

DAX 是 Power BI 和 SQL Server Analysis Services 中使用的函數語言，用於創建計算式、度值和表達式，以進行數據分析和建模。

規則說明範例
不分大小寫函數名稱、資料表和欄位名稱不區分大小寫SUM() 和 sum() 等效
使用括號函數必須使用括號包含參數SUM(Table[Column])
欄位引用欄位名稱用方括號包裹[SalesAmount]
資料表引用資料表和欄位名稱可以組合Table[SalesAmount]
運算符支持數學、邏輯和比較運算符+, -, *, >, AND, OR
行上下文DAX 函數在計算列時逐行執行新增資料行中的公式
篩選上下文量值會根據報表中的篩選動態計算量值隨篩選器自動變化

DAX 實作：新增案例數

實作

使用 CALCULATE + DATEADD 計算每日新增確診人數（今日累計 - 前一日累計）。

// 新增量值：新增案例數
新增案例數 =
VAR PreviousDayConfiremed =
    CALCULATE(
        SUM('confirmed'[確診人數]),
        DATEADD('confirmed'[日期], -1, DAY)
    )
RETURN
SUM('confirmed'[確診人數]) - PreviousDayConfiremed

DAX 實作：參數功能（動態排行）

進階

透過模型化功能建立參數，搭配 RANKX 實現動態 Top N 篩選。

步驟 1：建立參數

模型化 → 新增參數 → 名稱：排行 / 資料類型：整數 / 最小值 0 / 最大值 20 / 遞增 1

步驟 2：新增量值 — 總確診人數
總確診人數 = SUM(confirmed[確診人數])
步驟 3：新增量值 — 動態排行篩選
動態排行篩選 =
VAR SelectedRank = [排行 值]
VAR CurrentRank =
    RANKX(
        ALLSELECTED('confirmed'[Country/Region]),
        [總確診人數],
        ,
        DESC
    )
RETURN
IF(CurrentRank <= SelectedRank, [總確診人數], BLANK())

DAX 練習：產品銷售

實作

使用產品銷售.xlsx 練習各種 DAX 函數：

新增資料行 — 計算稅額
稅額 = '訂單'[銷售額] * 0.05
建立量值 — 計算總銷售額
總銷售額 = SUM('訂單'[銷售額])
建立資料表 — 篩選高於目標的銷售數據
超標銷售額 = FILTER('訂單', '訂單'[銷售額] > 150)
條件函數 IF — 額外折扣計算
額外折扣 = IF(
    '超標銷售額'[銷售額] > 1000,
    0.05 * '超標銷售額'[銷售額],
    0
)
時間函數 — 計算年度銷售額
年度銷售額 = TOTALYTD(
    SUM('超標銷售額'[銷售額]),
    '超標銷售額'[訂單日期]
)

合併格式相同的資料（附加）

實作

使用「附加查詢」將結構相同的多個資料表上下串接為單一表格（類似 SQL 的 UNION）。

  • 進入 Power Query 編輯器
  • 常用 → 附加查詢 → 「將查詢附加為新查詢」（不影響原始資料）
  • 可選擇兩個資料表或三（含）個以上的資料表
  • 範例：將 81-88、89-96、97-100 三個年度的死亡統計資料合併為一個表
提示：附加查詢會產生新的資料表（如「附加1」），不會影響原始資料。

合併不同格式的資料（合併查詢）

實作

使用「合併查詢」將不同結構的資料表依共同欄位橫向合併（類似 SQL 的 JOIN）。

  • 常用 → 合併查詢 → 選擇兩個資料表與對應的連結欄位
  • 範例：經緯度表（鄉鎮縣市、zip code、Latitude、Longitude）與死亡統計附加表（鄉鎮縣市(新)）合併
  • 聯結種類：
    • 左方外部（第一個的所有資料列、第二個的相符資料列）
    • 右方外部（第二個的所有資料列、第一個的相符資料列）
    • 完整外部、內部、左方反向、右方反向
  • 合併後需展開合併的欄位 → 點選展開按鈕 → 勾選要保留的欄位

資料夾匯入

實作

可從「取得資料」→「資料夾」一次匯入整個資料夾中的多個同格式檔案，自動合併。

進階圖表功能

環圈圖與群組分析

實作

以死亡統計資料為例，進行分組與視覺化分析。

  • 增加詳細資料：環圈圖可拖入「詳細資料」欄位（如死因分類），滑鼠懸停時顯示
  • 以量化新增群組：在資料表檢視中，右鍵欄位 → 群組 → 設定量化大小（如以 5 年為單位）
  • 以清單新增群組：右鍵 → 群組 → 群組類型「清單」→ 手動將鄉鎮歸入區域群組（如雙北基隆、中彰投、雲嘉南等）

小倍數與排序

實作
  • 新增小倍數效果：將分類欄位拖入圖表的「小倍數」區域，自動產生多個子圖表（如依死因分類分別顯示）
  • 排序功能：右鍵圖表 → 排序小倍數/排序軸 → 依特定欄位遞增或遞減排序
  • 自訂常數線：分析面板 → 常數線 → 輸入值（如 50000）作為比較基準線
  • 內建快速計算式：在值欄位右鍵 → 加總、平均、最小值、最大值、計數、標準差、變異數、中間值等
  • 快速量值：可建立每個類別的總和/平均/變異數/最大值/最小值/加權平均等

智慧分析功能

資訊
  • 問與答 (Q&A)：以自然語言提問生成圖表，目前不支援中文
  • 關鍵影響因素：自動分析特定數據的主要影響因素，提供對比分析（如「有什麼會影響死亡數減少？」）

連接資料庫 (MySQL)

MySQL 連接設定

實作
前置準備
  • 安裝 MySQL Connector/NET（建議版本 8.0.28，相容性較佳）
  • 安裝完成後需重新啟動 Power BI
連接步驟
  • 常用 → 取得資料 → 資料庫 → MySQL 資料庫
  • 填入伺服器位址（含 port），例如：dv108.aiturn.fun:3306
  • 填入資料庫名稱，例如：antdata_db
  • 輸入帳號密碼後即可選擇要匯入的資料表
