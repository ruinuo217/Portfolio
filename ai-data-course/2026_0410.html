協作開發與虛擬化技術 I：Docker 基礎

Docker 核心概念、Image/Container/Dockerfile、生命週期管理、Docker Hub 分享

Docker 解決什麼問題

情境：「在我電腦可以跑啊」

觀念

AI 工程師模型訓練好了，FastAPI 也包好了，但同事跑不起來：

  • CUDA 版本不對：你用 CUDA 12.1，他裝 11.8
  • Python 版本衝突：你用 3.11，他的系統是 3.9
  • 套件衝突：pip install 成功但 import 失敗（版本不相容）
  • OS library 缺失：libGL、libgthread 等系統套件沒裝
Docker 核心想法：不要叫別人裝環境，直接把裝好的環境給他。把程式碼 + 執行環境 + 系統套件全部打包成一個「容器」。

Docker vs 虛擬機（VM）

觀念

Docker 與傳統虛擬機的架構差異：

比較項目VM（虛擬機）Docker（容器）
虛擬化層級硬體層（Hypervisor）OS 層（共用 Kernel）
啟動速度分鐘級秒級
資源佔用GB 級（含完整 OS）MB 級（僅應用 + 依賴）
隔離性完全隔離行程級隔離
可攜性需搬整個 VM Image輕量 Image，秒推秒拉
典型用途需要不同 OS 的環境微服務部署 / CI/CD

Build → Ship → Run

觀念

Docker 的三階段工作流程：

  • Build（建構）：用 Dockerfile 定義環境 → docker build 產生 Image
  • Ship（交付）：把 Image 推上 Docker Hub → docker push
  • Run（執行）：任何機器拉下來就能跑 → docker pull + docker run

Docker 三大價值：

  • 環境一致：開發、測試、生產環境完全相同
  • 可重現：Dockerfile 就是環境的版本控制
  • 可交付：不是交程式碼，是交「能跑的環境」

Docker 安裝與設定

安裝步驟（Ubuntu）

實作

在 Ubuntu / Debian 系統上安裝 Docker：

# 1. 安裝 Docker
sudo apt update
sudo apt install docker.io -y

# 2. 啟動 Docker 服務
sudo systemctl start docker

# 3. 設定開機自動啟動
sudo systemctl enable docker

# 4. 確認服務狀態
sudo systemctl status docker

# 5. 確認版本
docker -v

權限設定

實作

預設只有 root 能執行 docker 指令，把使用者加入 docker 群組：

sudo usermod -aG docker $USER

指令逐段解說：

部分說明
sudo以管理員權限執行
usermod修改使用者帳號設定
-aG-a = append（附加），-G = 指定群組
docker要加入的群組名稱
$USER目前登入的使用者名稱
注意：執行後需要登出再登入（或 newgrp docker）才會生效。

Docker Image（使用者視角）

Image 是什麼

觀念

Docker Image 是唯讀的、包含一切執行所需的檔案系統快照。

  • 唯讀：Image 一旦建立就不會改變
  • 分層結構：每一層都是一個檔案系統變更
  • 包含一切：OS 基底 + 系統套件 + 程式碼 + 依賴

Image 與 Container 的關係：

  • Image = 設計圖（Class）：定義環境的模板
  • Container = 執行實例（Instance）：根據 Image 啟動的執行環境
  • 一個 Image 可以啟動多個 Container

Image 來源

觀念

取得 Image 的兩種方式：

  • Docker Hub（公開倉庫）：直接拉取官方或社群維護的 Image
  • 自行建立（Dockerfile）：用 Dockerfile 客製化自己的 Image

從 Docker Hub 取得 Image 的指令：

# 搜尋 Image
docker search python

# 下載 Image
docker pull python:latest

# 列出本機所有 Image
docker images

# 用 Image 啟動 Container（互動模式）
docker run -ti python:latest

docker run 旗標說明：

旗標說明
-t分配一個虛擬終端機（tty），讓輸出格式正常
-i保持 stdin 開啟，允許互動輸入

Workshop：下載並體驗 Image

Workshop

任務：從 Docker Hub 下載 Python Image 並進入互動模式。

  • 執行 docker pull python:latest 下載 Python Image
  • 執行 docker images 確認 Image 已下載
  • 執行 docker run -ti python:latest 進入 Python 互動模式
  • 在 Python Shell 中輸入 print("Hello Docker!") 測試
  • 輸入 exit() 離開 Container

Dockerfile（建構者視角）

Dockerfile 是什麼

觀念

Dockerfile 是一個純文字檔，描述如何一步步建構 Image。就像食譜一樣，記錄每一個步驟。

  • 檔名就叫 Dockerfile（沒有副檔名）
  • 每一行指令會產生一個新的層（Layer）
  • Docker 會從上到下依序執行每一行

FastAPI 最小版本

實作

main.py — FastAPI 應用程式：

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
    return {"message": "Hello Docker!"}

requirements.txt — 依賴清單：

fastapi
uvicorn

Dockerfile — 建構指令：

FROM python:3.11-slim
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

Dockerfile 逐行解說

觀念

每一行指令的作用與執行時機：

指令說明何時執行
FROM python:3.11-slim以 Python 3.11 精簡版為基底build 時
WORKDIR /app設定工作目錄（不存在會自動建立）build 時
COPY . .把本機檔案複製到 Container 的 /appbuild 時
RUN pip install ...安裝 Python 套件build 時
CMD ["uvicorn", ...]Container 啟動時執行的指令run 時

RUN vs CMD 差異

觀念

這是初學者最常混淆的兩個指令：

特性RUNCMD
執行時機docker build 時（建構階段）docker run 時（啟動階段）
執行次數可以有多個 RUN只有最後一個 CMD 生效
用途安裝套件、下載檔案、設定環境定義 Container 啟動後要執行的程式
結果產生新的 Image Layer不產生 Layer，只是記錄啟動指令

執行流程圖示：

Dockerfile
    |
    v
[docker build] ──── RUN pip install ... ──── 安裝套件（寫入 Image）
    |
    v
  Image（唯讀快照）
    |
    v
[docker run] ─────── CMD uvicorn ... ─────── 啟動應用程式（Container 內執行）
    |
    v
  Container（執行中）

Build 與執行

實作

建構 Image 並啟動 Container：

# 建構 Image（-t 指定名稱）
docker build -t my-fastapi .

# 確認 Image 已建立
docker images

# 啟動 Container（-p 對應 Port）
docker run -p 8000:8000 my-fastapi

瀏覽器打開 http://localhost:8000 即可看到回應。

Docker Container（生命週期）

Container 生命週期

觀念

Container 從建立到銷毀的完整流程：

create → start → running → stop → exited
  |         |         |          |         |
  |         |         |          |         +-- 已停止，可再 start
  |         |         |          +-- 發送停止信號
  |         |         +-- 正常運行中
  |         +-- 啟動 Container
  +-- 建立 Container（尚未啟動）

重點：docker run = create + start + run，一條指令完成三步。

為什麼用 0.0.0.0

觀念

FastAPI 為什麼要綁 0.0.0.0 而不是 127.0.0.1？

  • 127.0.0.1：只監聽 Container 內部的 loopback，外面連不進來
  • 0.0.0.0：監聽所有網路介面，允許外部（Host）透過 Port Mapping 連入

Port Mapping 流程：

Browser → Host:8000 → Container:8000 → FastAPI (0.0.0.0:8000)

docker run -p 8000:8000 my-fastapi
              |     |
              |     +-- Container 內部 Port
              +-- Host 對外 Port
記憶口訣：-p Host端口:Container端口，左邊是外面，右邊是裡面。

Docker Hub（Image 分享）

Docker Hub 是什麼

觀念

Docker Hub = 放 Image 的地方，就像 GitHub 放程式碼一樣。

  • 公開倉庫：任何人都可以拉取（如 python, nginx, ubuntu）
  • 私有倉庫：免費帳號可建立 1 個私有 Repository
  • 官方 Image：Docker 官方維護的高品質 Image

推送 Image 流程

實作

將本機 Image 推送到 Docker Hub 的完整步驟：

# 1. 註冊 Docker Hub 帳號（hub.docker.com）
# 2. 在 Docker Hub 新增 Repository

# 3. 本機登入
docker login

# 4. 為 Image 加上標籤（格式：帳號/Repository:標籤）
docker tag my-fastapi username/my-fastapi:latest

# 5. 推送到 Docker Hub
docker push username/my-fastapi:latest

# 6. 在其他機器上拉取
docker pull username/my-fastapi:latest
重點：docker tag 不會複製 Image，只是為同一個 Image 多加一個名稱（像 Git 的 tag）。

Workshop：上傳 Image 至 Docker Hub

Workshop

任務：將剛才建立的 FastAPI Image 推送到 Docker Hub。

  • 前往 hub.docker.com 註冊帳號（如果還沒有）
  • 建立一個新的 Repository（例如 my-fastapi）
  • 本機執行 docker login 輸入帳號密碼
  • 執行 docker tag my-fastapi <你的帳號>/my-fastapi:latest
  • 執行 docker push <你的帳號>/my-fastapi:latest
  • 到 Docker Hub 網頁確認 Image 已上傳

主線必學指令

必學

這 5 個指令可以處理 90% 的 Dockerfile 需求：

指令說明範例

WORKDIR設定工作目錄WORKDIR /app
COPY複製本機檔案到 ImageCOPY . .

進階查表指令

附錄

用到時再查即可，不需要背：

指令說明
ADD類似 COPY，但可解壓 tar 檔 / 下載 URL
ENV設定環境變數，如 ENV PORT=8000
EXPOSE宣告 Container 監聽的 Port（文件用途）
