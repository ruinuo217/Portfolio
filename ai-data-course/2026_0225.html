Oracle Analytics Desktop 簡介

Gartner BI 平台定位

觀念

在 Gartner 魔力象限中，Oracle 位於領導者與遠見者交界處，具備強大的 AI 與機器學習整合能力。在 MAD Landscape 中，Oracle Analytics Cloud 同時出現在 BI Platforms 與 BI & AI Analytics Platforms 分類。

安裝 ML 套件 (DVML)

實作

Oracle Analytics Desktop 需額外安裝機器學習和進階分析套件。

Windows 安裝步驟
  • Windows「開始」功能表 → 瀏覽至 Oracle → 點擊「安裝 DVML」
  • 系統會啟動終端機並下載 BiPython_Framework.exe
  • 安裝程式在下載完成時啟動，按指示安裝至選取路徑
  • 安裝完成後重新啟動 Oracle Analytics Desktop
Mac 安裝步驟
  • Finder → Oracle Analytics Desktop 應用程式資料夾 → 按兩下「Configure Python」
  • 系統下載 BiPython_Framework.pkg，輸入管理員密碼執行安裝

主頁與左側導覽

基礎

主控台管理對應（GeoJSON 地圖層）、擴充（視覺化外掛）、安全網域、工作簿翻譯

匯入與資料準備

實作
  • 建立 → 資料集 → 選擇 CSV 檔案上傳
  • 設定區隔符號（如 Tab 字元）、千分位符號（逗號）、小數區隔符號（句點）

資料流程與序列

資料流程 (Data Flow)

核心

資料流程是 Oracle Analytics 的 ETL（擷取、轉換、載入）功能，可對原始資料集進行篩選、合併、計算、分組，最後將結果儲存為新的乾淨資料集。

核心功能說明
彙總依據維度（如地區）新增計算式（如消費金額）
合併資料 (Join)將不同資料表合併
過濾器篩選資料
儲存資料整理後的資料匯出成新的數據源，供儀表板直接使用
資料流程步驟操作

左側選單提供豐富的資料處理步驟：

  • 新增資料、結合、資料列聯集、篩選、彙總、儲存資料集
  • 新增資料欄、選取資料欄、重新命名資料欄、轉換資料欄、合併資料欄、分割資料欄
  • 值格、群組、分支、累計值
  • 時間序列預測、分析情緒
  • 機器學習：訓練數值預測、訓練多重分類器、訓練分群法、訓練二元分類器、訓練自訂模型、套用模型

序列 (Sequence)

核心

序列用於設定多個資料流程的執行順序，確保前一個處理完畢後下一個才開始。

核心功能說明
批次執行依序觸發多個資料流程
流程相依性確保前一個資料處理完畢後，後一個才開始，避免數據不一致

資料流程 vs 序列

比較
功能資料流程序列
定位數據清洗設定流程的執行順序
處理對象原始數據欄位、表格已經做好的「資料流程」
產出物新的資料集執行完畢的狀態報告
範例情境把 1,000 筆顧客資料算出平均消費額先跑完「顧客清洗流程」，再跑「流失預測流程」

資料流程實作：顧客分群

步驟 1：建立資料分群流程

實作

以「顧客統計」資料為例，建立資料流程進行資料整理與分群。

流程：顧客統計 → 新增資料欄 → 儲存資料

新增「年齡區間」欄位
CASE
  WHEN 年齡 < 20 THEN '0-19'
  WHEN 年齡 < 30 THEN '20-29'
  WHEN 年齡 < 40 THEN '30-39'
  WHEN 年齡 < 50 THEN '40-49'
  WHEN 年齡 < 60 THEN '50-59'
  ELSE '60+'
END
新增「消費區間」欄位
CASE
  WHEN 消費金額 < 5000 THEN '0-4,999'
  WHEN 消費金額 < 20000 THEN '5,000-19,999'
  WHEN 消費金額 < 50000 THEN '20,000-49,999'
  ELSE '50,000+'
END

最後儲存資料，命名新資料集（如「顧客統計-整理後」），設定各欄位「視為」屬性或量值。

步驟 2：K-Means 分群模型訓練

進階

在資料流程中使用「訓練分群法」步驟，選擇 K-Means Clustering 模型。

K-Means 原理：找資料中心點，距離最近的分同一群。如：「把人群分成幾堆」。

流程：顧客統計-整理後 → 選取資料欄 → 訓練分群法 → 儲存模型

K-Means 參數設定
參數預設值說明
Number Of Clusters3群數（分成幾群）
Number Of Iterations300迭代次數（模型最多計算幾次），數字越大越準但耗時間
Number Of Initiation Runs10初始化次數（模型會跑幾次，選最佳結果）
Initialization MethodK-Means++初始化方式
Internal AlgorithmAuto內部演算法（用系統預設）
Categorical Column ImputationMost Frequent類別缺值處理（用數量最多的資料值補）
Numerical Column ImputationMean數值缺值（用平均值補）
Categorical Encoding MethodIndexer類別轉數值（模型只能使用數字，必須轉換）
Maximum Null Value Percent80允許缺值比例（超過 80% 空值的欄位會被忽略）
StandardizationTrue標準化（一定要開！所有欄位變成同一尺度）
Train Partition Percent80訓練比例（80% 訓練，20% 測試）

步驟 3：套用模型與建立序列

實作
  • 訓練完成後，在「機器學習」頁面可看到產生的模型（如「客戶分群」）
  • 建立使用模型的資料流程：
    • 新增資料流程 → 匯入整理後的資料 → 套用模型 → 儲存為新資料集
    • 流程：顧客統計-整理後 → 套用模型 → 儲存資料
  • 建立序列：將「顧客統計」和「使用分群模型」兩個資料流程排入序列，確保依序執行

練習題

練習

使用 air_traffic 資料建立資料流程：

  • 1. 設定時間資料型態正確
  • 2. 依 Landing Count 設定自訂分群
  • 3. 新產生資料檔使用 K-Means 模型訓練
  • 4. 新增資料流程：新資料集套用產生模型並產出套用後資料集
  • 5. 設定序列執行資料流程

BI 工具比較

Tableau vs Qlik Sense vs Power BI vs Oracle Analytics

觀念
工具優點缺點適合資料類型
Tableau 強大視覺化效果、支援多種數據來源與大數據 需要專業訓練、較缺乏 AI 分析功能 結構化 & 半結構化數據、視覺化報表、即時數據監控
Qlik Sense 自動發掘數據關聯性、即時數據分析與處理能力 介面學習曲線較陡、需學習 Qlik 語法、視覺化效果比 Tableau 稍弱 關聯性數據、多來源數據整合、IoT 即時監控、高複雜度數據
Power BI AI 分析功能強（Copilot、預測、異常偵測）、成本較低 數據處理能力較弱（超過 1GB 需 Premium）、視覺化自由度比 Tableau 低 結構化數據、企業報表與 KPI 監控、Microsoft 用戶
Oracle Analytics 內建 AI 與機器學習（趨勢預測、異常偵測）、與 Oracle Cloud 無縫整合、數據準備與 ETL 能力 非 Oracle 用戶較難適應、介面較傳統、小型企業使用門檻較高 企業級 AI 預測、金融、政府、Oracle ERP/Cloud 數據分析
如何選擇？
工具最適合情境
Tableau需要較佳的視覺化與探索數據
Qlik Sense處理大量數據，發掘關聯關係
Power BI中小企業，與 Microsoft 環境整合
Oracle Analytics大型企業，需要 AI 與預測分析
