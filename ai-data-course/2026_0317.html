YOLO 物件偵測與事件處理

3/17 課程筆記

YOLO 物件偵測概念

什麼是 YOLO

觀念

You Only Look Once — 一次看完整張影像就完成物件偵測。YOLO 以 CNN 為骨幹，不只判「是什麼」，還回答「在哪裡、多確定」。

  • 一次推論：一張影像同時找出多個物件及其位置，不需逐區掃描
  • 即時性：適合攝影機 / 串流場景，速度足以支撐即時處理
  • 結構化輸出：輸出的是 class / bbox / confidence，不是圖片
系統角色：YOLO 的定位是「影像 → 結構化觀測資料（detections）」。它把非結構化的圖片轉換成系統可以處理的結構化資料。

Pre-trained Models 選型

觀念

根據場景需求選擇適合的模型大小，先用小模型跑通流程，確定需求再換大模型。

模型大小速度適用場景
yolov5s小最快原型開發 / 教學
yolov5m中快一般系統
yolov5l大慢伺服器部署
yolov5x超大最慢高精度需求

Python 推論程式碼

實作

使用 ultralytics 套件載入模型並取得結構化輸出。

from ultralytics import YOLO

# 載入預訓練模型
model = YOLO("yolov5s.pt")

# 推論
results = model("image.jpg")

# 結構化輸出
for box in results[0].boxes:
    print(box.cls)    # class_id
    print(box.conf)   # confidence
    print(box.xyxy)   # bbox 座標 (x1, y1, x2, y2)

# 儲存結果圖
results[0].save()

觀念

解答：

訓練流程：

  • (1) Roboflow Universe 搜尋 helmet → 選 helmet-c1enh → Fork Dataset → 下載 YOLO 格式 ZIP

YOLO → Event JSON

為什麼要轉 Event

觀念

畫好框的圖片無法直接用於告警 / API 回傳 / 資料庫統計。系統需要的是結構化事件。

// 系統需要的格式
{
  "event_type": "helmet_detected",
  "confidence": 0.87,
  "bbox": {"x1": 120.5, "y1": 80.3, "x2": 340.1, "y2": 290.7},
  "timestamp": "2026-03-17T10:30:00Z"
}
核心觀念：YOLO 不是在「辨識安全帽」——YOLO 只是產生事件觸發條件。完整流程：影像 → YOLO 推論 → Detection Event → 規則 / API / 告警 / 紀錄。

標註轉事件實作

實作

save_txt=True 輸出 YOLO 格式 txt（class_id x_center y_center width height），再轉換為事件 JSON。

# label_to_event.py
# 讀取 YOLO 格式 txt → 組成事件 JSON
import json
from datetime import datetime

def parse_label(txt_path, image_name):
    events = []
    with open(txt_path) as f:
        for line in f:
            parts = line.strip().split()
            cls_id = int(parts[0])
            x_c, y_c, w, h = map(float, parts[1:])
            events.append({
                "event_type": f"class_{cls_id}_detected",
                "source_image": image_name,
                "bbox": {"x_center": x_c, "y_center": y_c,
                         "width": w, "height": h},
                "timestamp": datetime.utcnow().isoformat() + "Z"
            })
    return events

Workshop：圖片到事件

練習

題目：整合 YOLO 推論 + 事件轉換，輸入圖片輸出事件 JSON。

解答： 
from ultralytics import YOLO
from datetime import datetime
import json

LABEL_MAP = {0: "head", 1: "helmet"}

def detect_and_convert(model_path, image_path, min_confidence=0.5):
    model = YOLO(model_path)
    results = model(image_path)
    events = []
    for box in results[0].boxes:
        conf = float(box.conf[0])
        if conf < min_confidence:
            continue
        cls_id = int(box.cls[0])
        x1, y1, x2, y2 = box.xyxy[0].tolist()
        events.append({
            "event_type": f"{LABEL_MAP.get(cls_id,'unknown')}_detected",
            "confidence": round(conf, 4),
            "bbox": {"x1":round(x1,2),"y1":round(y1,2),
                     "x2":round(x2,2),"y2":round(y2,2)},
            "source_image": image_path.split("/")[-1],
            "timestamp": datetime.utcnow().isoformat() + "Z"
        })
    return events

events = detect_and_convert("best.pt", "test.jpg")
print(json.dumps(events, indent=2))

延伸思考：同一張圖多個 helmet → 每個 box 產生一筆事件。沒戴安全帽（class 0 = head）→ event_type = 'head_detected'。confidence 過濾在事件層做（模型層輸出所有偵測，事件層決定哪些值得處理）。

事件處理（去重 / 累積 / 計數）

為什麼需要事件處理

觀念

攝影機 30fps → 同一違規者 1 秒產生 30 筆事件 → 老闆手機 30 則推播。必須從 Frame-level 轉為 Event-level：30 筆 no_helmet（1 秒）→ 1 次違規。

操作說明範例
去重 Dedup合併重複偵測同一人連續影格只算一次
累積 Accumulate時間窗口統計過去 N 分鐘違規次數
計數 Count當前狀態場域內目前幾人

IoU 去重

觀念

IoU = 交集面積 / 聯集面積。IoU > 0.5 → 可能是同一物件。

  • EventDeduplicator：時間窗口內相同類別且 IoU > 閾值 → 視為重複 → 返回 None
  • 效果：30 筆 / 秒 → 1 筆事件

累積器與計數器

觀念

在去重後進一步統計與追蹤。

  • EventAccumulator：時間窗口（如 5 分鐘）內各類別事件數量。get_violation_count() 統計 no_helmet + no_vest
  • ObjectCounter：追蹤物件進入 / 離開，超時 5 秒未偵測 = 離開

事件處理管線

實作

EventPipeline：YOLO Events → 去重 → 累積 → 計數 → 輸出。

# EventPipeline.process() 回傳結構
{
    "is_new": True,        # 是否為新事件（通過去重）
    "violations": 3,       # 時間窗口內違規次數
    "person_count": 12     # 目前場域內人數
}

批次偵測與告警 Workshop

架構設計

觀念

FastAPI Server 接收圖片跑 YOLO 回傳事件 JSON + Client 逐一送圖片收集結果。

  • Server：接收上傳圖片 → YOLO 推論 → 組 event JSON → 回傳
  • Alert 判斷：有 head_detected = True（未戴安全帽）
  • Client：逐一上傳 images/ 目錄的圖片，所有結果寫 results.json

Server 端

實作

FastAPI UploadFile 接收圖片 → 存 temp.jpg → YOLO 推論 → 組 event JSON → 回傳。

from fastapi import FastAPI, UploadFile
from ultralytics import YOLO
import os

app = FastAPI()
model = YOLO("best.pt")
LABEL_MAP = {0: "head", 1: "helmet"}

@app.post("/detect")
async def detect(file: UploadFile):
    content = await file.read()
    with open("temp.jpg", "wb") as f:
        f.write(content)
    results = model("temp.jpg")
    events = []
    for box in results[0].boxes:
        cls_id = int(box.cls[0])
        events.append({
            "event_type": f"{LABEL_MAP[cls_id]}_detected",
            "confidence": round(float(box.conf[0]), 4),
            "bbox": box.xyxy[0].tolist()
        })
    has_alert = any(
        e["event_type"] == "head_detected" for e in events
    )
    os.remove("temp.jpg")
    return {
        "events": events,
        "event_count": len(events),
        "alert": has_alert
    }

Client 端

實作

requests.post 上傳圖片，收集 events 寫入 results.json。

import requests, json, os

all_events = []
for fname in sorted(os.listdir("images")):
    with open(f"images/{fname}", "rb") as f:
        r = requests.post(
            "http://localhost:8000/detect",
            files={"file": (fname, f, "image/jpeg")}
        )
    data = r.json()
    all_events.extend(data["events"])
    alert = " ALERT" if data["alert"] else ""
    print(f"{fname} -> {data['event_count']} events{alert}")

with open("results.json", "w") as f:
    json.dump(all_events, f, indent=2)

課程回顧

今日學習架構

觀念

完整流程：YOLO 安裝 → 推論（Pre-trained）→ 訓練（自訂模型）→ Event JSON（結構化輸出）→ 事件處理（去重 / 累積 / 計數）→ 批次偵測 + 告警（FastAPI + Client）。

核心觀念：YOLO 是感知層，負責產生事件，不是直接給答案。系統需要的是結構化資料，不是圖片。從影像到事件、從事件到決策，這才是完整的 AI 應用架構。
