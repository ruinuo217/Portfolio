Linux 進階操作與 Shell Script

從系統監控、權限管理到自動化腳本撰寫與資安設定。

Part 1: 虛擬機與網路原理

網路模式 (Network Modes)

觀念

VirtualBox 的網路設定差異。

  • NAT (預設)： 虛擬機透過主機上網，外部無法直接連入虛擬機。
  • Bridged (橋接介面)： 虛擬機視為區網內的獨立電腦，擁有獨立 IP，手機/電腦可直接連入。
  • Host-Only： 僅主機與虛擬機可互通，無法上網。

虛擬化軟體比較

知識
  • Windows/Linux： VMware Workstation, VirtualBox, Hyper-V (Microsoft)。
  • Mac OS： Parallels Desktop (付費且效能佳)。
  • Parrot OS： 這是一個資安滲透測試用的 Linux 系統，非虛擬機軟體。
注意： 同一台 Host 盡量不要同時開啟兩種 Hypervisor (如 VirtualBox + Hyper-V)，會導致搶佔硬體資源，效能大幅下降。

Part 2: 實作監控儀表板 (PHPSysInfo)

Apache + PHP 環境建置

實作

從無到有架設系統資訊監控網頁。

# 1. 安裝必要套件 (Apache, PHP, XML模組, 解壓縮工具)
sudo apt install apache2 php php-xml php-mysqli unzip -y

# 2. 進入網頁根目錄
cd /var/www/html/

# 3. 下載 phpsysinfo 原始碼
sudo wget https://github.com/phpsysinfo/phpsysinfo/archive/refs/tags/v3.4.4.zip

# 4. 解壓縮並更名資料夾
sudo unzip v3.4.4.zip
sudo mv phpsysinfo-3.4.4 phpsysinfo

# 5. 建立設定檔 (複製範例檔)
cd phpsysinfo/
sudo cp phpsysinfo.ini.new phpsysinfo.ini

# 6. 驗收
# 開啟瀏覽器輸入 IP/phpsysinfo 即可看到儀表板

移除與清理環境

維護

若要重做或移除服務的指令。

# 強制刪除網頁檔案
sudo rm -rf /var/www/html/*

# 移除軟體服務
sudo apt remove apache2 php php-xml unzip -y

# 自動清除不再需要的相依套件
sudo apt autoremove

Part 3: Shell Script 腳本設計

變數與基礎語法

Coding

Linux 中的變數設定。

# 設定變數 (等號兩邊不能有空白)
bob=168
echo $bob   # 輸出: 168

bob=8.8.8.8
echo $bob   # 輸出: 8.8.8.8

實作：ping.sh 互動腳本

Coding

撰寫一個讓使用者輸入網址並自動測試連線的腳本。

#!/bin/bash

# 詢問使用者輸入 (read -p)
read -p "請輸入網址 (Input URL): " url

# 執行 ping 指令 (-c 4 代表只跑4次)
ping -c 4 $url
執行步驟：
1. vim ping.sh (貼上程式碼)
2. chmod +x ping.sh (給予執行權限)
3. ./ping.sh (執行腳本)

Part 4: 系統資源與權限管理

檔案權限 (Permissions)

核心
  • chmod +x 檔名： 賦予檔案「執行 (Executable)」權限 (變綠色)。
  • chmod -x 檔名： 移除「執行 (Executable)」權限 (變回一般顏色)。
  • ls -l / ll： 查看詳細權限 (rwx)。

系統監控指令

工具
  • top / htop： 即時查看 CPU 與記憶體使用量 (按 q 離開)。
  • free -h： 查看記憶體 (RAM) 剩餘量。
  • df -h： 查看硬碟 (Disk) 剩餘空間。
  • cal： 顯示月曆 (如 cal 2025)。

Part 5: 資安與日誌管理

日誌分析 (Logs)

資安

路徑：/var/log/

  • auth.log： 紀錄登入驗證資訊 (包含 SSH 登入失敗紀錄)。
  • head auth.log：看前 10 行。
  • tail -n 5 auth.log：看最後 5 行。
  • grep "關鍵字" auth.log：搜尋特定內容 (如 accepted)。
  • watch tail -n 5 auth.log：即時監控日誌變化。

帳號與 SSH 設定

危險
  • su root： 切換到超級管理員 (權限過大，操作需小心)。
  • sudo passwd root： 修改 root 密碼。
  • 修改 SSH Port：
    • 編輯設定檔：sudo vim /etc/ssh/sshd_config
    • 將 Port 22 改為其他數字 (如 8022) 以避免惡意掃描。
    • 修改後需重啟服務 (擇一)：
      1. sudo service ssh restart
      2. sudo reboot
