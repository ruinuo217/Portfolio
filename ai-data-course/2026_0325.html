事件規則引擎與 ML 分群

3/25 課程筆記 — YAML 規則引擎、N 幀聚合、Pandas 事件統計、KMeans 非監督分群、資料驅動門檻優化

Rule-based 事件判斷

從 hard-coded 到可配置

觀念

if label=="head": has_alert=True 的問題：每次改規則要改 code + 重新部署。解法：YAML 設定檔，把規則從程式碼中抽離。

  • 規則格式：name / event_type / min_confidence / max_confidence / action (alert / warning / ignore / ok) / severity
  • 比對邏輯：由上往下比對，第一條符合就生效
  • 設計原則：低信心忽略放最前面，確保 confidence ≤ 0.5 的不會誤觸 alert

YAML 規則範例

實作

4 條規則，由上往下依序比對：

#名稱event_type信心度範圍actionseverity
1低信心忽略-max: 0.5ignore-
2沒戴安全帽高信心head_detectedmin: 0.7alerthigh
3沒戴安全帽低信心head_detected-warninglow
4預設--ok-

alert_threshold: 2 — 連續 2 張圖有 alert 才發佈（避免單幀誤判）。

規則比對邏輯

實作

check_event(event, rules)：逐條檢查 event_type / min_confidence / max_confidence，全部通過則回傳該 rule。

  • 逐條比對：for rule in rules，第一條全部條件都通過就回傳
  • 條件檢查：event_type 不符 → continue、confidence 不在範圍 → continue
  • N 幀聚合：統計每張圖 alert 數，連續 N 張都有 alert 才發佈（避免單幀誤判）

Workshop：規則引擎

練習

完成 check_event / process_events / aggregate_alerts 三個函式。

完整解答： 
import yaml, json

def check_event(event, rules):
    for rule in rules:
        if "event_type" in rule and event["event_type"] != rule["event_type"]:
            continue
        if "min_confidence" in rule and event["confidence"] < rule["min_confidence"]:
            continue
        if "max_confidence" in rule and event["confidence"] > rule["max_confidence"]:
            continue
        return rule
    return None

def process_events(events, rules):
    results = []
    for event in events:
        matched = check_event(event, rules)
        event["action"] = matched["action"] if matched else "unknown"
        event["rule"] = matched["name"] if matched else "none"
        results.append(event)
    return results

def aggregate_alerts(results, threshold):
    alert_images = set()
    for r in results:
        if r["action"] == "alert":
            alert_images.add(r["source_image"])
    return len(alert_images) >= threshold, list(alert_images)

規則順序很重要：低信心忽略放第一條，確保 confidence ≤ 0.5 的不會誤觸 alert。N 幀聚合避免單幀誤判：mixed1.jpg 有 5 筆 alert + mixed2.jpg 有 1 筆 = 2 張 ≥ 門檻 2 → 發佈 ALERT。

Sklearn KMeans 分群

為什麼用 KMeans 找門檻

觀念

rules.yaml 的 min_confidence: 0.7 怎麼來的？人工看幾張圖拍腦袋。KMeans 自動從資料分佈找分群。

方法人工設定門檻KMeans 分群
依據看幾張圖的感覺全部資料分佈
客觀性主觀可重複驗證
擴展性看不完大量資料自動處理

演算法地圖：有標籤 → 監督學習 (KNN)，沒標籤 → 非監督學習 → 分群 → KMeans。

KMeans 三步驟

觀念

KMeans 演算法核心流程：

  • Step 1：隨機放 K 個中心點
  • Step 2：每筆資料歸到最近中心
  • Step 3：重新計算中心 → 重複直到穩定

範例 K=3 結果：低信心群（中心 0.62）/ 中信心群（中心 0.76）/ 高信心群（中心 0.84）。切分點：低/中 ≈ 0.70，中/高 ≈ 0.81。

監督 vs 非監督

觀念

兩種學習方式的比較：

比較項目監督學習 KNN非監督學習 KMeans
標籤需求需要標籤不需要標籤
目的預測新資料類別找資料自然分群
適用情境有歷史標註想發現模式

我們的情境：沒人標過高/中/低 → 用 KMeans。

實作：KMeans 分群

實作

使用 sklearn 進行 KMeans 分群的完整流程：

  • from sklearn.cluster import KMeans
  • X = df[["confidence"]].values — 取出信心度欄位
  • kmeans = KMeans(n_clusters=3, random_state=42).fit(X)
  • cluster_centers_ 取得中心點 → 排序 → 算切分點
  • 更新 rules.yaml 的門檻值

Workshop：KMeans 找門檻

練習

用 KMeans 自動找出信心度的高/中/低切分點。

完整解答： 
from sklearn.cluster import KMeans
import pandas as pd, json
import numpy as np

# 讀取事件資料
with open("rule_results.json") as f:
    df = pd.DataFrame(json.load(f))
X = df[["confidence"]].values

# KMeans分3群
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
kmeans.fit(X)
df["cluster"] = kmeans.labels_

# 取得中心點並排序
centers = sorted(kmeans.cluster_centers_.flatten())
print(f"群中心: {[round(c,4) for c in centers]}")
# 例如: [0.5535, 0.7502, 0.8438]

# 計算切分點(相鄰中心的平均)
thresholds = [(centers[i]+centers[i+1])/2 for i in range(len(centers)-1)]
print(f"建議門檻: {[round(t,4) for t in thresholds]}")
# 例如: [0.6519, 0.7970]

# 各群統計
for i in range(3):
    cluster_data = df[df["cluster"]==i]["confidence"]
    print(f"群{i}: n={len(cluster_data)}, range=[{cluster_data.min():.4f}, {cluster_data.max():.4f}]")

KMeans 找出的門檻比人工更客觀：基於全部資料分佈而非幾張圖的感覺。K=3 代表分成低/中/高三群。切分點 = 相鄰群中心的平均值。這些門檻可直接更新到 rules.yaml：第一個切分點 → max_confidence（忽略門檻），第二個切分點 → min_confidence（alert 門檻）。

HW 完整答案

練習

串起所有模組的完整整合程式碼：

完整整合程式碼： 
import pandas as pd, json, yaml
from sklearn.cluster import KMeans
import numpy as np

# === Step 1: Pandas 統計原始資料 ===
with open('results.json') as f:
    df = pd.DataFrame(json.load(f))
print("=== 原始資料統計 ===")
print(f"總筆數: {len(df)}")
print(df["event_type"].value_counts())
print(f"信心度: mean={df['confidence'].mean():.4f}, std={df['confidence'].std():.4f}")

# === Step 2: KMeans 找門檻 ===
X = df[["confidence"]].values
kmeans = KMeans(n_clusters=3, random_state=42, n_init=10).fit(X)
centers = sorted(kmeans.cluster_centers_.flatten())
thresholds = [(centers[i]+centers[i+1])/2 for i in range(len(centers)-1)]
print(f"\n=== KMeans 結果 ===")
print(f"群中心: {[round(c,4) for c in centers]}")
print(f"建議門檻: {[round(t,4) for t in thresholds]}")

# === Step 3: 設計 YAML 規則 ===
rules_config = {
    "rules": [
        {"name":"低信心忽略","max_confidence":round(thresholds[0],2),"action":"ignore"},
        {"name":"沒戴安全帽","event_type":"head_detected","min_confidence":round(thresholds[1],2),"action":"alert","severity":"high"},
        {"name":"沒戴安全帽低信心","event_type":"head_detected","action":"warning","severity":"low"},
        {"name":"預設","action":"ok"}
    ],
    "alert_threshold": 2
}
with open('auto_rules.yaml','w') as f:
    yaml.dump(rules_config, f, allow_unicode=True)
print(f"\n規則已寫入 auto_rules.yaml")

# === Step 4: 規則引擎處理 ===
def check_event(event, rules):
    for rule in rules:
        if "event_type" in rule and event.get("event_type") != rule["event_type"]:
            continue
        if "min_confidence" in rule and event.get("confidence",0) < rule["min_confidence"]:
            continue
        if "max_confidence" in rule and event.get("confidence",1) > rule["max_confidence"]:
            continue
        return rule
    return {"name":"unknown","action":"unknown"}

results = []
for _, row in df.iterrows():
    event = row.to_dict()
    matched = check_event(event, rules_config["rules"])
    event["action"] = matched["action"]
    event["rule"] = matched["name"]
    results.append(event)
result_df = pd.DataFrame(results)

# === Step 5: Pandas 分析結果 ===
print(f"\n=== 規則比對結果 ===")
print(result_df["action"].value_counts())
alerts = result_df[result_df["action"]=="alert"]
print(f"\nalert事件: {len(alerts)}筆")
if len(alerts) > 0:
    print(alerts[["source_image","event_type","confidence","rule"]].to_string())
    alert_images = alerts["source_image"].unique()
    should_alert = len(alert_images) >= rules_config["alert_threshold"]
    print(f"\n有alert的圖片: {list(alert_images)}")
    print(f"{'發佈ALERT' if should_alert else '不發佈'}(門檻:{rules_config['alert_threshold']})")

這串起了所有模組：Pandas 統計 → KMeans 找門檻 → YAML 規則自動生成 → 規則引擎處理 → Pandas 分析結果。關鍵理解：KMeans 讓門檻從「拍腦袋」變成「資料驅動」，但最終門檻仍需人工審核調整 —— ML 是建議，人是決策者。
