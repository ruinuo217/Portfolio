網路爬蟲基礎與 requests

從 robots.txt 協議、HTTP 請求原理，到 BeautifulSoup 解析與 DOM 遍歷策略。

Part 1: 爬蟲概念與規範 (Concept & Robots.txt)

爬蟲的角色與限制

觀念
什麼是網路爬蟲 (Web Scraper)?

自動從網路上搜集並擷取數據的技術，是 AI 學習語言模式、市場趨勢的重要資料來源 (Offline Data Provider)。

⚠️ 關鍵架構觀念：離線 vs 線上
不要在「即時系統 (Runtime)」中直接執行爬蟲！
  • 錯誤： 用戶點擊 -> 系統去爬網站 -> 回傳結果 (網站掛了系統就掛了)。
  • 正確： 爬蟲 (Offline) -> 存成快照 (laws.json) -> 系統讀取快照 (Runtime)。
robots.txt 協議

網站與爬蟲的溝通橋樑，放置於網站根目錄 (e.g., apple.com/robots.txt)。它是一種「君子協定」，非強制性安全機制。

規則語法範例說明
User-agentUser-agent: Googlebot指定適用的爬蟲 (* 代表所有爬蟲)。
DisallowDisallow: /admin/禁止訪問的路徑。
AllowAllow: /public/允許訪問的路徑 (通常用於 override Disallow)。
# 範例：禁止所有爬蟲進入 private 資料夾
User-agent: *
Disallow: /private/

Part 2: HTTP 協定與 Requests (HTTP Protocol)

HTTP 請求結構

基礎
  • Request (請求):
    • Method: GET (查詢), POST (提交)
    • Headers: User-Agent, Authorization
    • Body: 攜帶的資料 (JSON/Form)
  • Response (回應):
    • Status Code: 200 (成功), 404 (未找到)
    • Body: HTML 原始碼或 JSON 資料

Python Requests 實戰

實作

安裝: pip install requests

GET 與 POST 請求
import requests

# GET: 獲取資料
res = requests.get("https://httpbin.org/get")
print(res.text)

# POST: 帶 Header 與 Data
headers = {"User-Agent": "MyBot/1.0"}
data = {"name": "Reyna"}

res = requests.post(
    "https://httpbin.org/post", 
    headers=headers, 
    json=data
)

Part 3: BeautifulSoup 解析 (Parsing HTML)

BeautifulSoup 查找神器

核心
基本操作

安裝: pip install beautifulsoup4

from bs4 import BeautifulSoup

html = "<a href='...'>Link</a>"
soup = BeautifulSoup(html, 'html.parser')

# 取得第一個匹配的標籤
tag = soup.a 
print(tag.text) # 文字內容
print(tag['href']) # 取得屬性
find vs find_all
方法回傳用途
find()Tag 物件找「第一個」符合的。
find_all()ResultSet (List)找「所有」符合的。
進階篩選
# 找 id="link1" 的 a 標籤
soup.find_all("a", id="link1")

# 找含有特定文字的標籤
soup.find_all("a", string="Elsie")

Part 4: DOM Traversal (樹狀結構遍歷)

DOM 移動方法

進階

當無法直接定位時，從「鄰居」或「父母」出發。

  • .parent: 往上找父節點。
  • .children: 往下找子節點。
  • .next_sibling: 下一個兄弟 (小心空白換行也是節點)。
  • .previous_sibling: 上一個兄弟。
Tag 是物件： 找到的 Tag 可以繼續 call .find()，不一定要從 soup 開始找。

定位策略

策略
策略 1: By Attribute (推薦)

利用 id, class, data-* 這些語意明確的屬性當作錨點 (Anchor)。

# 先抓有 id 的容器，再往下找 p
container = soup.find("div", id="container")
target = container.find("p")
策略 2: By String (最後手段)

結構亂且無屬性時，用「顯示文字」定位，再推算位置。

# 找到 "台灣時間：" 的標籤，再找它的父層的下一個兄弟
label = soup.find("p", string="台灣時間：")
time_div = label.parent.previous_sibling
