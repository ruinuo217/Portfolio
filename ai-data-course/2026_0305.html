Unit 01 - GCP 平台架構

雲端運算三種服務模式

觀念

雲端服務依照使用者管理範圍的不同，分為三種模式。

模式 全稱 GCP 代表服務 使用者負責
IaaS 基礎架構即服務 Compute Engine / Cloud Storage 管理 OS 與應用程式
PaaS 平台即服務 App Engine / Cloud Run 只需部署程式碼
SaaS 軟體即服務 Gmail / Google Workspace 瀏覽器直接使用

GCP 服務分類地圖

觀念

GCP 將服務依功能分為多個核心類別，以下為常用分類。

類別 代表服務
運算 Compute Engine / GKE / Cloud Run
儲存 Cloud Storage / Persistent Disk
網路 VPC / Load Balancing
資料庫 Cloud SQL / Firestore
管理 IAM / Cloud Monitoring

專案與資源層級

觀念

GCP 資源以階層結構組織：組織 → 資料夾 → 專案 → 資源。

  • 專案 ID：全球唯一，建立後不可更改
  • 專案名稱：可隨時修改
  • 專案編號：系統自動產生的數字 ID

GCP 帳號設定實作

啟用免費試用

實作

GCP 提供 $300 美元 / 90 天免費試用額度，不會自動扣款。

  • 前往 Google Cloud 官網 → 開始免費使用
  • 填寫帳單資訊（信用卡驗證不扣款）

建立第一個專案

實作

在 Console 頂部專案選擇器建立新專案。

  • Console 頂部專案選擇器 → 新增專案
  • 填寫專案名稱 / ID / 位置

Console 介面導覽

實作

熟悉 GCP Console 的三大區域。

  • 頂部導覽列：專案選擇 / 搜尋 / Cloud Shell
  • 左側選單：服務分類
  • 主要內容區：服務操作介面
  • 釘選 Compute Engine / Cloud Storage / IAM 到導覽列

練習：GCP 帳號設定

練習

啟用 GCP 免費試用並完成初始設定。

優秀回答： 成功啟用 GCP 免費試用並建立專案，Console 右上角確認 Region 與專案名稱。Cloud Shell 執行 gcloud config list 顯示當前專案 ID。將 Compute Engine、Cloud Storage、IAM 釘選到導覽選單頂部，方便後續操作。專案 ID 建議加入日期或個人識別碼避免重複，例如 gcp-learn-20260305。

常見錯誤排查

觀念

帳號設定常見問題與解決方式。

問題 解決方式
專案配額上限 免費帳號限 12 個，刪除不用的專案
找不到服務 需先啟用對應的 API
帳單驗證失敗 確認信用卡支援國際交易

IAM 權限管理

IAM 三大要素

觀念

IAM（Identity and Access Management）由三大要素組成。

  • 主體（Principal）：Google 帳號 / 服務帳號 / 群組
  • 角色（Role）：Owner / Editor / Viewer / 預先定義 / 自訂
  • 資源（Resource）：專案 / 服務 / 單一資源
權限繼承： 組織 → 資料夾 → 專案 → 資源，子層級繼承父層級權限且無法移除。

基本角色 vs 預先定義角色

觀念

基本角色權限範圍較廣，預先定義角色可精細控制。

基本角色 權限 適用對象
Viewer 只能檢視 稽核人員
Editor 可建立、修改、刪除 開發人員
Owner 完整控制（含 IAM） 專案管理員
  • 預先定義角色範例：compute.admin、storage.objectViewer 等
注意： 基本角色權限過大，不建議在生產環境使用，應改用預先定義角色。

服務帳號

觀念

供應用程式或 VM 使用的特殊帳號。

  • 格式：name@project-id.iam.gserviceaccount.com
  • 建議用 GCP 管理金鑰（VM 自動附加）
  • 避免下載 JSON 金鑰（容易外洩）

練習：IAM 操作

練習

檢視 IAM 設定、建立服務帳號 demo-vm-service-account（授予 Storage Object Viewer）。

優秀回答： 在 IAM 頁面看到我的 Google 帳號為 Owner，還有 Compute Engine 和 App Engine 的預設服務帳號。建立了 demo-vm-service-account 並授予 roles/storage.objectViewer，遵循最小權限原則——只給需要的讀取權限。服務帳號優先使用 VM 附加方式，避免下載 JSON 金鑰，若金鑰外洩需立即停用並檢查 Cloud Logging 異常存取。

Compute Engine 虛擬機

機器類型選擇

觀念

依照工作負載選擇合適的機器系列。

系列 特性 適用場景
E2 成本優化，e2-micro 免費方案可用 學習用
N2 穩定效能 生產環境
C2 運算優化 CPU 密集型工作
M2 記憶體優化 記憶體內資料庫
  • 學習 → e2-micro
  • 小型 Web → e2-small / e2-medium
  • 生產環境 → n2-standard-2 起

磁碟類型

觀念

開機磁碟（含 OS，必須指定）vs 永久磁碟（可獨立於 VM）。

類型 IOPS 成本 適用場景
Standard PD (HDD) 低 最低 冷資料
Balanced PD (SSD) 中 中 一般應用
SSD PD 高 高 資料庫
Extreme PD 極高 最高 關鍵系統

連線方式比較

觀念

Compute Engine VM 支援多種連線方式。

方式 特色 需求 適用場景
瀏覽器 SSH 最簡易 點擊即連 快速檢視
gcloud CLI 可自動化 需安裝 SDK 腳本化操作
標準 SSH 進階功能 需管理金鑰 本機整合
Windows RDP 圖形介面 需設密碼 Windows Server

SSH 金鑰登入

觀念

使用 SSH 金鑰對進行安全登入。

  • ssh-keygen -t ed25519 生成金鑰對
  • 公鑰加入 GCP Metadata SSH Keys
  • 私鑰登入：ssh -i key ubuntu@IP
  • 原理：公鑰加密驗證

VM 建置實作

建立第一個 VM

實作

建立 Compute Engine 虛擬機實例。

  • 名稱：demo-web-server
  • 區域：us-central1-a
  • 類型：e2-micro
  • OS：Ubuntu 22.04 LTS
  • 磁碟：10GB Balanced PD
  • 防火牆：允許 HTTP + HTTPS

SSH 連線與部署 Nginx

實作

透過瀏覽器 SSH 連線並安裝 Web Server。

  • 瀏覽器 SSH 連線
  • apt update
  • apt install nginx
  • echo 自訂首頁內容
  • 瀏覽器打開 VM 外部 IP 驗證

防火牆規則

實作

建立允許 HTTP 流量的防火牆規則。

  • 名稱：allow-http-from-internet
  • 方向：輸入（Ingress）
  • 目標：所有執行個體
  • 來源：0.0.0.0/0
  • Port：TCP 80

gcloud CLI 管理

實作

使用 gcloud 指令管理 VM 實例。

  • gcloud compute instances list
  • gcloud compute instances start [NAME]
  • gcloud compute instances stop [NAME]
  • gcloud compute instances delete [NAME]
  • gcloud compute ssh [NAME]

練習：VM 部署驗證

練習

驗證 VM 建立成功並確認 Nginx 部署正常。

優秀回答： VM 建立成功，外部 IP 為 34.x.x.x，瀏覽器打開看到 Hello from GCP! 頁面。架構：Internet → VM(Compute Engine) → SSH:22(管理) + HTTP:80(對外)。hostname 顯示 demo-web-server，whoami 顯示使用者名稱，cat /etc/os-release 確認 Ubuntu 22.04。若無法存取網站，排查順序：(1) systemctl status nginx 確認服務 (2) curl localhost 測本機 (3) 確認防火牆規則有開 TCP:80 (4) 確認使用的是外部 IP 而非內部 IP(10.x.x.x)。

資源管理與成本控制

資源標籤

觀念

鍵值對（Key-Value）用於分類、追蹤成本和自動化。

  • 成本追蹤：department:marketing
  • 自動化：auto-shutdown:true
  • 資源清查：data-classification:confidential
標籤 vs 網路標記： 標籤用於管理與帳單分類，網路標記用於防火牆流量控制。

預算警示

實作

設定預算避免超額支出。

  • 設定每月預算金額
  • 警示門檻：50% / 90% / 100%
  • Email 通知

刪除資源完整流程

觀念

刪除 VM 後需逐一檢查所有相關資源。

  • VM 執行個體
  • 永久磁碟
  • 外部 IP
  • 快照
  • 映像檔
  • 防火牆規則
注意： 刪除 VM 不等於刪除所有相關資源！靜態 IP 即使未使用也計費，永久磁碟未附加也計費。

練習：資源管理

練習

為 VM 加標籤、建預算警示、正確刪除 VM。

優秀回答： 為 demo-web-server 加入 environment=development 和 owner=student 標籤，可在帳單報表依標籤分組分析成本。設定 $50/月預算，警示門檻 50%/90%/100%。刪除 VM 後還需檢查：gcloud compute disks list（未附加磁碟）、gcloud compute addresses list（靜態 IP）、gcloud compute snapshots list（快照）。標籤資料同步到帳單需 24-48 小時。

Unit 02 - Cloud Storage 核心概念

為什麼用 Cloud Storage

觀念

相比傳統虛擬主機：空間無限制 / 自動擴展 / 免維護 / 內建 CDN / 支援生命週期管理。

需求 建議服務
圖片、影片、備份 Cloud Storage
靜態網站 Cloud Storage
VM 硬碟 Persistent Disk
SQL 資料 Cloud SQL

三大核心概念

觀念

Bucket、Object、Storage Class 是 Cloud Storage 的三大核心。

  • Bucket：雲端倉庫，全球唯一名稱，建立後不可改名
  • Object：實際檔案，路徑如 gs://bucket/css/style.css
Storage Class 定位 存取頻率 成本
Standard 熱資料 頻繁存取 最貴
Nearline 溫資料 每月 1-2 次 省 50%
Coldline 冷資料 每季 1 次 省 80%
Archive 封存 幾乎不存取 最便宜
注意： Cloud Storage 實際是扁平結構，「資料夾」只是檔案名稱的一部分。

Cloud Storage 實作

建立 Bucket

實作

建立 Cloud Storage Bucket 的設定要點。

  • 命名規則：小寫 / 數字 / 連字號，3-63 字元
  • 位置選擇：Region asia-east1（成本低 / 個人網站用）vs Multi-region ASIA（高可用 / 企業用）
  • 儲存類別：Standard
  • 存取控制：統一（Uniform）

上傳與管理檔案

實作

透過 Console 或 gsutil 指令管理檔案。

  • Console 上傳：單檔 / 資料夾 / 拖放
  • gsutil cp：上傳/下載檔案
  • gsutil ls：列出檔案
  • gsutil rm：刪除檔案
常見錯誤： 中文檔名可能造成問題（建議用英文），Content-Type 錯誤時需檢查 MIME type。

設定公開存取

實作

將 Bucket 設為公開可讀。

  • 權限分頁 → 授予存取權
  • 主體：allUsers
  • 角色：Storage Object Viewer
安全風險： 任何人都能看所有檔案，不要放 API 金鑰 / 密碼 / 用戶資料在公開 Bucket。

靜態網站託管

實作

使用 Cloud Storage 託管靜態網站。

  • 編輯網站設定：Index page = index.html
  • 404 page = 404.html
  • 網址：https://storage.googleapis.com/bucket-name/index.html

練習：Cloud Storage 操作

練習

建立 Bucket、上傳檔案、設定公開存取與靜態網站。

優秀回答： 建立 Bucket 時選 asia-east1（台灣）降低延遲，命名全球唯一如 my-portfolio-2026-03。上傳 index.html/css/images 後設定 allUsers=Storage Object Viewer 公開存取。靜態網站託管設定 index.html 為主頁。四種 Storage Class 差異：Standard 適合頻繁存取的網站檔案；Nearline 適合每月偶爾存取的舊作品（省 50%）；Coldline 適合備份（省 80%）；Archive 適合法規保存（最便宜但取回慢）。Bucket 是扁平結構不是資料夾，存取控制選 Uniform 統一管理最簡單。

生命週期管理與成本優化

自動儲存類別轉換

觀念

設定生命週期規則自動降級儲存類別以節省成本。

  • 90 天後 → Nearline（省 50%）
  • 365 天後 → Coldline（省 80%）
  • 也可設定自動刪除（如暫存檔 30 天後刪除）
  • 每 24 小時執行一次，不立即生效
注意： 刪除是永久性的，建議先在測試 Bucket 驗證規則。

進階：自訂域名架構

觀念

透過 HTTPS Load Balancer + Backend bucket 實現自訂域名。

  • 架構：使用者 → https://www.myportfolio.com → GCP HTTPS LB → Backend bucket → Cloud Storage bucket
  • 優點：自訂域名 / 自動 HTTPS / CDN 加速 / 不需 VM

整合思考題

GCP vs AWS 概念對照

觀念

兩大雲端平台的服務對照表。

AWS GCP
EC2 Compute Engine
S3 Cloud Storage
Security Group 防火牆規則
IAM User IAM 主體
EBS Persistent Disk
AMI 映像檔
CloudFront Cloud CDN

思考題

練習

你的公司要建立一個內部知識庫網站，需要存放文件（PDF）和影片教學，請設計儲存方案。

優秀回答： (1) 建立兩個 Bucket：docs-bucket（PDF 文件，Standard，Region asia-east1）和 videos-bucket（影片，Standard 轉 Nearline）。(2) 權限：不用 allUsers 公開，改用 IAM 群組控制（如 knowledge-readers@company.com 授予 Storage Object Viewer）。(3) 生命週期：影片 90 天後轉 Nearline（省 50%），365 天後轉 Coldline（省 80%）；PDF 文件保持 Standard（頻繁查閱）。(4) 成本估算：100GB PDF + 500GB 影片，Standard 約 $12/月，套用生命週期後降至 $5/月。(5) 與 VM 方案比較：VM 需 $25+/月且要維護，Cloud Storage 免維護且可搭配 Cloud CDN 加速。
