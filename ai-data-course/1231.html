MariaDB 資料庫與 PHP 連線實作

SQL 環境建置、Adminer 管理工具部署與時區校正。

Part 1: MariaDB 資料庫安裝

安裝與啟動

基礎

在 Ubuntu 上架設資料庫服務。

# 安裝 MariaDB Server
sudo apt install mariadb-server -y

# 啟動服務 (非必要，通常安裝後會自啟)
sudo systemctl start mariadb

# 檢查 Port 3306 是否開啟
netstat -tnpl

安全性設定 (Secure Install)

核心

執行 sudo mysql_secure_installation 進行初始化。

  • Set root password? [Y] 設定資料庫 root 密碼 (注意：非 Linux 系統 root)。
  • Remove anonymous users? [Y] 移除匿名登入。
  • Disallow root login remotely? [Y] 禁止遠端 root 登入 (安全性考量)。
  • Remove test database? [Y] 移除測試資料庫。
  • Reload privilege tables? [Y] 重新載入權限。

Part 2: PHP 連線與除錯

建立測試頁面

實作

建立 /var/www/html/db/index.php 測試連線。

# 1. 建立資料夾
cd /var/www/html
sudo mkdir db
cd db

# 2. 編輯 PHP 檔案
sudo vim index.php
常見錯誤：HTTP ERROR 500
原因：PHP 看不懂 MySQL 指令 (缺少模組)。
解法：sudo apt install php-mysqli
裝完後重新整理網頁即可正常運作。

PHP 連線範例碼

Coding

簡單的資料庫讀取與 UI 優化。

<?php
  // 設定時區
  date_default_timezone_set('Asia/Taipei');
  
  // 資料庫連線設定
  $servername = "localhost";
  $username = "root";
  $password = "你的密碼";
  $dbname = "diary";

  // 建立連線
  $conn = new mysqli($servername, $username, $password, $dbname);

  // 檢查連線
  if ($conn->connect_error) {
    die("Connection failed: " . $conn->connect_error);
  }
  echo "Connected successfully";
?>

Part 3: Adminer 管理工具

安裝 Adminer

工具

輕量級的資料庫管理介面 (替代 phpMyAdmin)。

# 1. 下載 Adminer (單一 PHP 檔案)
sudo wget https://github.com/vrana/adminer/releases/download/v5.4.1/adminer-5.4.1.php

# 2. 更名 (方便輸入網址)
sudo mv adminer-5.4.1.php adminer.php

# 3. 登入使用
# 網址：http://你的IP/db/adminer.php
# 帳號：root / 密碼：(剛剛設定的密碼)

Part 4: 系統校正與補充

系統時區校正

維護

解決資料庫時間 (Time) 與實際時間不符的問題。

# 查看目前時間
date

# 設定時區為台北
sudo timedatectl set-timezone Asia/Taipei

# 重啟資料庫讓設定生效
sudo systemctl restart mariadb

補充：osCommerce

專案

開放原始碼的電商購物車系統。

  • 版本：osCommerce Online Merchant v2.3.3.4
  • 需配合 LAMP 環境 (Linux, Apache, MySQL, PHP) 運作。
  • 資料庫設定需透過 Adminer 建立專用 DB。
