Linux 系統與架站實務

指令操作、Apache 伺服器架設、CMS 部署與資料庫管理

Chapter 1 & 2: Linux 基礎與指令

檔案與目錄操作

基礎

在終端機中移動與管理檔案。

  • pwd：顯示目前所在位置。
  • ls -al .：列出目前目錄下所有檔案 (含隱藏檔)。
  • cd ..：回到上一層目錄；~ 代表家目錄。
  • cp 來源 目的：複製檔案。
  • mv 來源 目的：移動或重新命名檔案。
  • rm -rf：強制刪除檔案與資料夾 (小心使用)。

連線與系統管理

管理

遠端連線與帳號權限。

  • SSH 連線： ssh root@IP
  • 查詢 IP： ifconfig 或 ip a
  • 帳號管理： adduser (新增), passwd (改密碼)。
  • 電源管理： shutdown -h 0 (關機), reboot (重啟)。
  • 編輯器： vi 或 vim (文字編輯器之戰)。

檔案下載與解壓

工具

從網路獲取資源至伺服器。

  • wget URL：從網址下載檔案。
  • unzip 檔案.zip：解壓縮 zip 檔。
  • FileZilla： 透過 SFTP (Port 22) 上傳檔案至 /var/www/html/。

Chapter 3: 網頁伺服器架設 (LAMP)

Apache 與 PHP 建置

核心

建立 Web Server 環境。

  • 安裝 Apache： sudo apt install apache2
  • 服務控制： sudo service apache2 start / restart
  • 安裝 PHP： sudo apt install php (讓 server 看懂 php)。
  • 安裝模組： apt install php-xml (某些 CMS 需要 DOM 支援)。

PHP 資訊頁測試

測試

確認 PHP 模組是否運作正常。

Command:
sudo vim /var/www/html/phpinfo.php
Code:
<?php
  phpinfo();
?>

CMS 安裝實作：Bludit

應用

輕量級內容管理系統部署。

  • 下載： wget 取得 bludit zip 檔並解壓。
  • 權限設定： sudo chmod 777 bludit/bl-content (方便但危險)。
  • 網址轉換： sudo a2enmod rewrite (啟用 Apache rewrite 模組)。
  • 權限歸屬： chown -R www-data:www-data bludit (將檔案交給 Apache 管理)。

實作坑點：PHP 版本相容性

除錯

Nibbleblog 安裝問題 (Ubuntu 24.04 預設 PHP 8.1)。

  • 問題： Nibbleblog 舊版不支援 PHP 8，需降版至 PHP 7.4。
  • 解法： 加入 PPA 儲存庫 (ppa:ondrej/php)。
  • sudo add-apt-repository ppa:ondrej/php
  • sudo apt install php7.4

Chapter 4: 資料庫與管理

MariaDB & Adminer

資料庫

資料庫系統與圖形化管理工具。

  • 安裝 DB： sudo apt install mariadb-server
  • 啟動 DB： sudo systemctl start mariadb
  • Adminer： 單一 PHP 檔案的輕量級資料庫管理工具 (類似 phpMyAdmin)。
  • 操作： 登入 Adminer 建立自己名字的資料庫。
