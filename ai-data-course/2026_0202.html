Pandas 資料分析基礎

AI 系統的記憶體：從即時事件 (Event) 到長期行為分析 (Behavior Analysis)。

Part 1: Pandas 在 AI 系統的角色 (Role & Concept)

多模態 AI 系統架構

觀念

為什麼需要 Pandas？因為 LLM 無法直接閱讀數萬筆原始 Log，Pandas 負責將「原始事件」轉化為「有意義的行為摘要」。

層級工具任務時間維度
感知層YOLO / Sensors看見什麼 (Detection)毫秒 (ms)
反應層Rule / State Machine即時反應 (Alert)秒 (s)
記憶層Pandas長期紀錄與彙總日/週/月
認知層LLM (GPT/Gemini)解釋與建議 (Explain)回顧
核心觀念： Pandas 是 AI 專案的「記憶體」。沒有 Pandas，AI 系統就只有「當下」，沒有「歷史」。

Part 2: DataFrame 基礎與結構 (Structure)

DataFrame 核心概念

基礎
  • 定義： 多個 Series 共用同一組 Index 的二維表格。
  • Row (列)： 代表一個「事件」(Event)。
  • Column (欄)： 代表事件的「屬性」(Attribute)。
import pandas as pd

data = {
    "event_id": [101, 102],
    "confidence": [0.92, 0.76]
}
df = pd.DataFrame(data)

事件表標準欄位 (Schema)

設計

AI 系統的最小可用欄位設計：

欄位意義
timestamp行為時間軸 (最重要的骨架)。
person行為主體 (User, Camera, Worker)。
event行為類型 (Login, Enter_zone)。
value數值 (Score, Count)。

Part 3: 資料處理與清洗 (Cleaning & Processing)

時間處理 (Time)

必做

字串時間無法計算，必須轉型。

# 轉型為 datetime64[ns]
df["timestamp"] = pd.to_datetime(df["timestamp"])

# 計算時間間隔
df["delta"] = df["timestamp"].diff()

缺值處理 (Missing Data)

清洗

資料進模型前必須處理 NaN。

# 1. 檢查哪裡有缺值
df.isna()

# 2. 補值 (例如補 0)
df["value"] = df["value"].fillna(0)

# 3. 刪除缺值 (當資料不可信時)
df_clean = df.dropna()

Part 4: 分析與彙總 (Aggregation)

Groupby 與 統計

分析
基本彙總
# 統計每個人有幾筆事件
df.groupby("person").size()

# 統計哪種 event 最常出現
df["event"].value_counts()
衍生欄位 (Derived Column)

透過計算產生新特徵，供 AI 判斷。

# 簡單計算
df["risk_score"] = df["fail_count"] * 2

# 複雜邏輯用 apply
df["level"] = df["score"].apply(risk_func)
