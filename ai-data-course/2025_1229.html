計算機概論：資料表示與軟體

深入理解 0 與 1 的世界：從數字系統、多媒體編碼到程式語言。

Part 1: 數字系統與資料表示 (Number Systems)

類比 vs 數位

觀念

到底數位是什麼呢？

  • 類比 (Analog)： 連續變化的物理量 (聲音、溫度)。圖形為波形。
  • 數位 (Digital)： 不連續數值 (0 與 1)。圖形為矩形波。
  • 優點： 抗干擾強、易於儲存處理。
  • 單位： Bit (位元)、Byte (8 bits)、Word (字組, CPU處理單位)。

數字系統轉換

運算

人類用十進位，電腦用二進位。

常見進位：
2進位 (0,1)
8進位 (0-7)
10進位 (0-9)
16進位 (0-9, A-F)
轉換邏輯：
二轉十：乘上權重 ($2^n$) 相加。
十轉二：短除法 (除以2取餘數)。

整數表示法 (Integers)

核心

電腦如何儲存正負數？

  • 帶符號 (Signed)： 最高位 (MSB) 0正 1負。
  • 一補數 (1's Complement)： 0 變 1，1 變 0 (反相)。
  • 二補數 (2's Complement)： 電腦標準。一補數 + 1。解決了 "+0" 與 "-0" 問題。

浮點數 (Floating Point)

進階

IEEE 754 標準：儲存含小數點的實數。

符號
指數
假數 (尾數)
  • 符號 (Sign)： 0正 1負。
  • 指數 (Exponent)： 數值大小範圍。
  • 假數 (Mantissa)： 數值精確度 (需先正規化)。

Part 2: 多媒體編碼 (Multimedia)

文字編碼 (Text)

編碼

將文字符號轉為二進位。

  • ASCII： 美國標準，7或8 bits (英數符號)。
  • Big5： 台灣繁體中文標準 (2 bytes)。
  • Unicode (萬國碼)： 解決亂碼，UTF-8 為網頁主流。
  • 格式： TXT, DOCX, PDF, ODF。

圖形與影像 (Graphics)

視覺
類型比較
  • 點陣圖： 像素組成，放大失真 (JPG, PNG)。
  • 向量圖： 數學公式，放大清晰 (SVG, AI)。
色彩模式
  • RGB： 螢幕顯示 (光三原色)。
  • CMYK： 印刷輸出。
關鍵指標：解析度 (PPI/DPI)、色彩深度 (Bit Depth)

聲音與視訊 (Audio/Video)

訊號

類比轉數位的過程。

  • 聲音轉數位： 取樣 (Sampling) → 量化 (Quantization)。
  • 取樣頻率： Hz 越高越真實。
  • 視訊原理： 視覺暫留。
  • FPS： 幀率，24 FPS 以上較流暢。

Part 3: 軟體與程式語言 (Software)

軟體類型分類

架構

軟體 = 程式 + 文件。

  • 系統軟體： 管理硬體 (OS, 驅動, 編譯器, 公用程式)。
  • 應用軟體： 滿足特定需求 (Office, 瀏覽器, LINE)。
  • ODF (開放文件格式)： 高互通性、利於長久保存 (LibreOffice)。

程式語言演進 (Programming Languages)

歷史

從 0/1 到自然語言的進化。

世代 名稱 特徵
第一代 機器語言 0與1組成，電腦直接執行，具機器依賴性。
第二代 組合語言 使用助憶符號 (Mnemonics) 代替機器碼。
第三代 高階語言 接近人類語言 (C, Java, Python)。需編譯/直譯。
第四代 極高階語言 特定領域 (SQL)。
執行流程：原始程式 → 編譯/組譯 → 目的程式 → 連結 → 執行檔
