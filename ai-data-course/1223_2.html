ESP-01S 物聯網入門

硬體認識、環境架設與 Blink 程式實作

Part 1: 硬體認識 (Hardware)

ESP-01S 基礎介紹

硬體

認識你手上的開發裝置。

  • ESP-01S (MCU)： 上層黑色小板子，微控制器（大腦）。
  • USB 轉 TTL： 下層 USB 插頭，負責電腦與晶片間的訊號翻譯。
  • 燒錄模組 (CH340C)： 支援「自動燒錄免跳線」，插上電腦即可寫入程式。

版本規格辨識

硬體

如何區分 ESP-01 舊版與 ESP-01S 新版？

  • 看燈號： 舊版通電紅燈亮；01S (新版) 只有傳輸時藍燈閃 (省電)。
  • 看 PCB： 舊版藍色；01S 為黑色。
  • 記憶體： 舊版 512KB；01S 為 1MB (可寫複雜程式)。

腳位定義與圖解

圖表

重點腳位說明：
1. VCC / GND：電源 (3.3V，不可接 5V)。
2. TXD / RXD：傳輸資料與燒錄用。
3. GPIO0 / GPIO2：控制用的手腳 (接繼電器或感測器)。

IoT 應用場景：一片小板子的無限可能

應用
  • 1. 智慧家電控制
    利用 MQTT 協定與手機 App (MQTTBox)，遠端遙控家中冷氣開關。
  • 2. 環境監測 (視覺化)
    使用 CO2 感測器偵測空氣品質，上傳至 ThingSpeak 繪製折線圖 (監測昏睡指數)。
  • 3. 智慧農業 (Blynk)
    土壤濕度感測器 + 抽水馬達。透過 Blynk 手機 App 監控數值並一鍵澆水。
  • 4. 自動化警報 (IFTTT)
    結合光線感測器或氣象資料。設定「若有人闖入」或「明天會下雨」，自動發送 LINE/Gmail 通知。

Part 2: 軟體開發 (Software)

Arduino IDE 介紹

工具

主要功能與優勢。

  • 三大功能： 撰寫 (Coding)、編譯 (Verify)、上傳 (Upload)。
  • 優勢： 跨平台免費、擁有豐富的函式庫 (Libraries) 免去造輪子。
  • 序列埠監控 (Serial Monitor)： 除錯神器，可看晶片回傳的數據。

環境設定 SOP

步驟

ESP8266 開發板安裝步驟圖解。

1. 設定開發板網址 (Preferences)
  1. 開啟 File > Preferences。
  2. 在 "Additional Boards Manager URLs" 貼上 ESP8266 網址。
2. 選擇開發板 (Board Manager)
  • 確認 Board 選擇 Generic ESP8266 Module。

程式碼解析：Blink (閃爍 LED)

實作
Step 1: 開啟範例 (Examples > ESP8266 > Blink)
Step 2: 程式碼預覽

邏輯解析：讓內建 LED 亮 1 秒，滅 2 秒。

核心觀念：Active Low (低電位觸發)
LOW (0V) = 電流通過 = 亮燈 (ON)
HIGH (3.3V) = 無電位差 = 滅燈 (OFF)
  • void setup()： 只執行一次。設定 pinMode(LED_BUILTIN, OUTPUT);
  • void loop()： 無限循環。
  • digitalWrite(LED_BUILTIN, LOW); // 開燈 (注意是 LOW)
  • delay(1000); // 等待 1 秒
  • digitalWrite(LED_BUILTIN, HIGH); // 關燈
  • delay(2000); // 等待 2 秒
